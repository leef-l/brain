## L0：资金自适应（Auto-Scaling）✅

### 问题

当前所有风控参数都是绝对值或固定百分比，换个资金量就得手动改一遍配置。

### 实现

`auto_risk` 配置段，根据账户实际权益自动计算风控参数。用户只需设置风险偏好等级，系统自动推导所有具体数值。

### 配置

```yaml
auto_risk:
  enabled: true
  level: aggressive    # conservative / moderate / aggressive
  min_order_usdt: 5.0
```

```

max_concurrent          3             8           15
stop_new_trades_loss    2%            5%          10%
liquidate_all_loss      3%            8%          15%
kelly_scale           0.15          0.25         0.35
max_fraction            3%            5%           8%
────────────────────────────────────────────────────────────
```

## L1：统计学习（Strategy Adaptation）✅

### 1.1 策略权重自适应

**文件**: `brains/quant/learning/weight_adapter.go`

根据近 N 笔交易各策略的实际贡献，动态调整 aggregator 权重。

```
每个策略维护一个"有效性分数" = 近 N 笔中该策略信号方向与最终盈亏一致的比例

新权重 = 基础权重 × (0.5 + 有效性分数)

约束：
  - 最小权重 = 基础权重 × 0.3
  - 最大权重 = 基础权重 × 2.0
  - 归一化后使用
  - 最少 30 笔交易才开始调整
```

**运行周期**: 每 5 分钟自动更新

### 1.2 品种偏好评分

**文件**: `brains/quant/learning/symbol_scorer.go`

```
品种评分 = 胜率 × 0.4 + 利润因子归一化 × 0.3 + 交易频率 × 0.3
未充分交易的品种返回 0.5（中性）
```

### 1.3 止损/止盈优化

**文件**: `brains/quant/learning/sl_tp_optimizer.go`

基于历史 MAE/MFE 分布分析，优化 SL/TP ATR 倍数。

```
SL 建议 = P75(MAE 分布) → 转为 ATR 倍数，clamp [1.0, 5.0]
TP 建议 = P50(MFE 分布) → 转为 ATR 倍数，clamp [1.5, 8.0]
当 P75(MFE) > 1.8 × P50(MFE) 时，建议使用移动止盈
```

**应用方式**: 70% 策略原始值 + 30% 优化器建议值混合

### 1.4 MAE/MFE 追踪

**文件**: `brains/quant/brain.go` (trackMAEMFE 方法)

每个价格 tick 更新持仓的 MAE（最大不利偏移）和 MFE（最大有利偏移），写入 TradeRecord。

### 1.5 TradeRecord 扩展字段

```go
type TradeRecord struct {
    // ... 基础字段 ...
    Leverage   int     // 实际杠杆倍数
    StopLoss   float64 // 入场时 SL 价格
    TakeProfit float64 // 入场时 TP 价格
    ATR        float64 // 入场时 ATR 值
    Confidence float64 // 聚合信号置信度
    Strategy   string  // 主导策略名称
    MAE        float64 // 最大不利价格偏移
    MFE        float64 // 最大有利价格偏移
}
```

PG migration 自动添加这些列（幂等 ALTER TABLE ADD COLUMN IF NOT EXISTS）。

## L2：ML 特征增强 ✅

特征向量 `[176:192]` 的 ML 增强段：

| 维度 | 含义 | RuleFallback | StatisticalMLEngine | HTTPMLEngine |
|------|------|-------------|---------------------|-------------|
| [176:180] | 市场 regime 概率 | ADX/EMA/BB 规则 | HMM 贝叶斯状态转移 | 神经网络分类 |
| [180:184] | 波动率预测 | ATR 外推 | EWMA 短期/长期 | 时序回归 |
| [184:188] | 异常检测 | 固定阈值 | Z-score 在线检测 | Isolation Forest |
| [188:192] | 预留 | 0 | 0 | 未来扩展 |

### 三层 ML 引擎降级链

```
HTTPMLEngine (外部推理服务)
    ↓ 不可用时自动降级
StatisticalMLEngine (纯 Go 统计引擎)
    ↓ 冷启动期（前 50 次 Update）
RuleFallback (规则启发式)
```

### 2.1 StatisticalMLEngine（当前生产运行）

**文件**: `brains/data/feature/statistical_engine.go`

纯 Go 实现，零外部依赖，启动 ~50 秒后自动就绪。

**核心算法**:
- **MarketRegime**: 4 状态隐马尔可夫转移矩阵 + 贝叶斯更新
- **VolPredict**: 短期(5) / 长期(20) EWMA + 百分位排名
- **AnomalyScore**: Welford 在线算法 + Z-score（2σ 开始计分，3σ 满分）

```go

```

### 2.2 HTTPMLEngine（代码就绪，待部署推理服务）

**文件**: `brains/data/feature/http_engine.go`

通过 HTTP 调用外部 Python 推理服务，支持 ONNX/PyTorch 模型。

**特性**:
- 请求超时 500ms（推理必须快）

- 失败时自动回退到 StatisticalMLEngine

### 步骤二：创建推理服务

创建文件 `/opt/brain-ml/server.py`：

```python
"""
Brain ML 推理服务
接收 176 维规则特征，输出 16 维 ML 增强特征。

协议：
  POST /predict  — 推理
  GET  /health   — 健康检查
"""
import os
import numpy as np
from fastapi import FastAPI
from pydantic import BaseModel

app = FastAPI(title="Brain ML Inference")

# ---------- 模型加载 ----------

# 模型文件路径（.onnx 格式）
MODEL_DIR = os.environ.get("MODEL_DIR", "/opt/brain-ml/models")

# 全局模型变量
regime_model = None      # 市场 regime 分类 [176:180]
vol_model = None         # 波动率预测 [180:184]
anomaly_model = None     # 异常检测 [184:188]

def load_models():
    """尝试加载 ONNX 模型。模型不存在时返回 None（静默降级）。"""
    global regime_model, vol_model, anomaly_model
    try:
        import onnxruntime as ort
        regime_path = os.path.join(MODEL_DIR, "regime_classifier.onnx")
        vol_path = os.path.join(MODEL_DIR, "vol_predictor.onnx")
        anomaly_path = os.path.join(MODEL_DIR, "anomaly_detector.onnx")

        if os.path.exists(regime_path):
            regime_model = ort.InferenceSession(regime_path)
            print(f"Loaded regime model: {regime_path}")
        if os.path.exists(vol_path):
            vol_model = ort.InferenceSession(vol_path)
            print(f"Loaded vol model: {vol_path}")
        if os.path.exists(anomaly_path):
            anomaly_model = ort.InferenceSession(anomaly_path)
            print(f"Loaded anomaly model: {anomaly_path}")
    except ImportError:
        print("onnxruntime not installed, using fallback")
    except Exception as e:
        print(f"Model loading error: {e}")

load_models()

# ---------- 请求/响应模型 ----------

class PredictRequest(BaseModel):
    features: list[float]  # 176 维规则特征

class PredictResponse(BaseModel):
    market_regime: list[float]  # [4] P(trend), P(range), P(breakout), P(panic)
    vol_predict: list[float]    # [4] vol1H, vol4H, volPercentile, volDirection
    anomaly_score: list[float]  # [4] price, volume, orderbook, combined
    reserved: list[float]       # [4] 预留

# ---------- 端点 ----------

@app.get("/health")
def health():
    """健康检查。Go 端 HTTPMLEngine 每 30 秒 ping 一次。"""
    return {"status": "ok", "models": {
        "regime": regime_model is not None,
        "vol": vol_model is not None,
        "anomaly": anomaly_model is not None,
    }}

@app.post("/predict", response_model=PredictResponse)
def predict(req: PredictRequest):
    """
    推理端点。
    输入: 176 维规则特征向量
    输出: 16 维 ML 增强特征（4 组 × 4 维）
    """
    features = np.array(req.features, dtype=np.float32).reshape(1, -1)

    # --- Regime 分类 ---
    if regime_model is not None:
        regime_out = regime_model.run(None, {"input": features})[0][0]
        market_regime = regime_out.tolist()
    else:
        # 无模型时返回均匀分布
        market_regime = [0.25, 0.25, 0.25, 0.25]

    # --- 波动率预测 ---
    if vol_model is not None:
        vol_out = vol_model.run(None, {"input": features})[0][0]
        vol_predict = vol_out.tolist()
    else:
        vol_predict = [0.0, 0.0, 0.5, 0.0]

    # --- 异常检测 ---
    if anomaly_model is not None:
        anomaly_out = anomaly_model.run(None, {"input": features})[0][0]
        anomaly_score = anomaly_out.tolist()
    else:
        anomaly_score = [0.0, 0.0, 0.0, 0.0]

    return PredictResponse(
        market_regime=market_regime,
        vol_predict=vol_predict,
        anomaly_score=anomaly_score,
        reserved=[0.0, 0.0, 0.0, 0.0],
    )
```

## 反循环保护（Anti-Churn）

**问题**: 在 1 秒评估周期 + signal_exit 下，信号闪烁会导致开仓→1秒平仓→开仓的死循环，每轮纯亏手续费。

**解决方案**:

| 参数 | 默认值 | 作用 |
|------|--------|------|
| `min_hold_duration` | 30s | 开仓后至少持有 30 秒才允许 signal_exit 平仓 |
| `cooldown_after_exit` | 60s | signal_exit 平仓后同品种 60 秒内不再开新仓 |

**配置**:

```yaml
signal_exit:
  enabled: true
  min_confidence: 0.4
  require_multi_strategy: 1
  min_hold_duration: 30s
```

## 风险与约束

1. **过拟合风险**：样本量少时统计结论不可靠 → 设 MinSamples 门槛，样本不足用基础值
2. **regime 切换**：过去的统计不代表未来 → 用滑动窗口（近 N 笔），而非全历史
3. **自动调参上限**：自动计算的值不能突破手动配置的天花板 → `min(auto, manual)` 约束
4. **学习速度**：不能每个 tick 都重新计算 → 策略权重 5 分钟更新一次，风控参数权益变化 >10% 才更新
5. **回退机制**：任何学习模块异常时回退到基础配置，不影响正常交易
6. **ML 引擎降级**：HTTPMLEngine → StatisticalMLEngine → RuleFallback，三级自动降级
7. **内存约束**：Python 推理服务需要额外 1-2GB 内存，小内存服务器（<4GB）不建议部署
