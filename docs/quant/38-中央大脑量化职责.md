# 中央大脑量化职责

> 监督者 · 审查者 · 复盘分析师 — 不参与交易决策

**上级文档**: [35-量化系统三脑架构总览](35-量化系统三脑架构总览.md)

---

## 一、核心定位

```
Central Brain ≠ 交易决策者
Central Brain = 监督者 + 审查者 + 协调者 + 复盘分析师
```

Central Brain 在量化系统中**不参与正常交易路径**。它只在以下场景被触发：

| 职责 | 触发条件 | 频率 |
|------|---------|------|
| LLM 交易审查 | 高风险信号 (NeedsReview) | ~几次/天 |
| 每日复盘 | UTC 0:00 定时 | 1 次/天 |
| 策略权重调整 | 复盘后 / 手动 | 1 次/天 |
| 异常处理 | 数据质量告警、账号故障 | 按需 |
| 多账号监控 | 定时轮询 | 每 5 分钟 |
| 紧急干预 | 极端行情、全局止损触发 | 极少 |
| 宏观事件解读 | 重大新闻/数据发布 | 按事件 |

## 二、LLM 交易审查

### 2.1 触发条件

Quant Brain 的 `NeedsReview(plan)` 返回 true 时，通过 JSON-RPC 工具调用 `specialist.call_tool -> central.review_trade` 进入 Central Brain：

- 持仓数 ≥ `trigger_concurrent`（默认 3）
- 最大单仓占比 > `trigger_position_pct`（默认 5%）
- 今日累计亏损 > `trigger_daily_loss`（默认 3%）
- 异常市场状态（AnomalyDetected）

### 2.2 审查流程

```
Quant Brain                    Kernel                     Central Brain
    │                             │                              │
    │  specialist.call_tool       │                              │
    │  target=central             │                              │
    │  tool=central.review_trade  │                              │
    │  {signal, candidates,       │                              │
    │   portfolio, trigger}       │                              │
    │────────────────────────────►│  tools/call                  │
    │                             │─────────────────────────────►│
    │                             │                         构造 Prompt
    │                             │                         调用 Claude Haiku
    │                             │                         解析 JSON 响应
    │                             │◄─────────────────────────────│
    │  ToolCallResult             │                              │
    │  {approved, size_factor,    │                              │
    │   reason}                   │                              │
    │◄────────────────────────────│                              │
    │                             │                              │
    │  超时/错误 → SizeFactor=1.0 fallback                       │
```

## 三、每日复盘

### 3.1 触发时间

UTC 0:00（北京时间 8:00），由 Central Brain 内部定时器触发。

### 3.2 输入数据

Central Brain 通过 JSON-RPC 从 Data Brain 和 Quant Brain 收集：

```go
type DailyReviewInput struct {
    Date             string

    // 从 Quant Brain 收集
    AccountPnL       map[string]AccountDailyPnL  // 各账号今日表现
    Traces           []SignalTrace                // 今日所有信号追踪
    StrategyStats    map[string]StrategyDailyStat // 策略级统计
    TotalTrades      int
    TotalWins        int
    TotalPnLPct      float64

    // 从 Data Brain 收集
    MarketSummary    MarketDailySummary           // 今日市场回顾
    DataQualityStats DataQualityDailyStats        // 数据质量统计
}

type AccountDailyPnL struct {
    AccountID string
    Label     string
    Mode      string
    Equity    float64
    PnLUSDT   float64
    PnLPct    float64
    Trades    int
    Wins      int
    MaxDD     float64
}

type StrategyDailyStat struct {
    Strategy  string
    Trades    int
    Wins      int
    PnLPct    float64
    AvgHold   time.Duration
    Signals   int      // 生成的信号数
    Rejected  int      // 被风控拒绝的数

}

type MarketDailySummary struct {
    BTCChange24h   float64
    ETHChange24h   float64
    MaxVolatility  float64
```

### 3.5 复盘建议动作

```go
type ReviewAction struct {
    Type        string         // "adjust_weight", "pause_strategy", "alert"
    Params      map[string]any
    AutoExecute bool           // true = 自动执行，false = 需人工确认
    Reason      string
}

// 示例动作:
// {
//   Type: "adjust_weight",
//   Params: {strategy: "MeanReversion", current: 0.25, suggested: 0.15},
//   AutoExecute: false,
//   Reason: "连续3天亏损，均值回归策略在趋势市中表现不佳"
// }
```

### 3.6 动作执行规则

| 动作类型 | 自动执行 | 说明 |
|---------|---------|------|
| `adjust_weight` | **否** | 权重调整需人工确认 |
| `pause_strategy` | **否** | 暂停策略需人工确认 |
| `alert` | 是 | 仅记录告警 |
| `reduce_leverage` | **否** | 降低杠杆需人工确认 |

**原则**: 涉及交易行为变更的动作一律需要人工确认。只有纯信息类动作自动执行。

### 3.7 复盘持久化

复盘结果存入 `daily_reviews` 表，包含：
- 输入数据 JSON
- 分析结果 JSON
- 建议动作 JSON
- 是否已执行
- 耗时

## 四、异常处理

### 4.1 数据质量告警

Data Brain 的 DataValidator 检测到异常时，通过 JSON-RPC 通知 Central Brain：

```go
func (c *CentralBrain) handleDataAlert(ctx context.Context, alert ValidatorAlert) {
    switch alert.Level {
    case "critical":
        // 价格 spike、数据源完全断连
        // → 通知所有人（日志 + 指标）
        // → 可能暂停对应品种交易
        if alert.Type == "price_spike" {
            c.callTool("quant.pause_instrument", map[string]any{
                "symbol": alert.Symbol,
                "reason": alert.Detail,
            })
        }
    case "warning":
        // 数据延迟、小规模 gap
        // → 仅记录，不干预
    }
}
```

### 4.2 账号故障

Quant Brain 检测到某账号执行器异常时，通过 `specialist.call_tool -> central.account_error` 通知 Central Brain：

```go
func (c *CentralBrain) handleAccountError(ctx context.Context, accountID, error string) {
    // 1. 记录告警
    // 2. 检查是否需要紧急平仓（如果是 live 账号）
    // 3. 通知人工介入
}
```

### 4.3 极端行情

Circuit Breaker 触发时的处理链路：

```
Quant Brain Circuit Breaker 触发
    → 所有账号暂停新交易
    → 通知 Central Brain
    → Central Brain 用 LLM 分析当前市场
    → 决定: 继续暂停 / 部分恢复 / 全部恢复
    → 通知人工（如果是重大事件）
```

## 五、多账号监控

Central Brain 每 5 分钟通过 `quant.global_portfolio` 工具轮询全局状态：

```go
func (c *CentralBrain) monitorAccounts(ctx context.Context) {
    view := c.callTool("quant.global_portfolio")

    // 检查异常
    for _, acct := range view.AccountSummaries {
        if acct.Status == "error" {
            c.handleAccountError(ctx, acct.ID, "status=error")
        }
        if acct.PnLToday < -3.0 {
            c.alert("warning", fmt.Sprintf("账号 %s 今日亏损 %.1f%%", acct.Label, acct.PnLToday))
        }
    }

    // 检查全局
    if view.TotalPnLToday < -5.0 {
        c.alert("critical", fmt.Sprintf("全局日亏损 %.1f%%，可能触发全局止损", view.TotalPnLToday))
    }
}
```

## 七、HandleMethod 汇总

```go
func (c *CentralBrain) Tools() []string {
    return []string{
        "central.review_trade",
        "central.data_alert",
        "central.account_error",
        "central.macro_event",
    }
}

func (c *CentralBrain) HandleMethod(ctx context.Context, method string, params json.RawMessage) (interface{}, error) {
    switch method {
    case "tools/call":
        return c.handleToolCall(ctx, params)   // central.review_trade / central.data_alert / central.account_error / ...
    case "brain/execute":
        var req sidecar.ExecuteRequest
        json.Unmarshal(params, &req)
        switch req.Instruction {
        case "daily_review_run":
            return c.dailyReview(ctx, req.Context)
        case "health_check":
            return c.healthCheck(ctx, req.Context)
        case "emergency_action":
            return c.emergencyAction(ctx, req.Context)
        case "update_config":
            return c.updateConfig(ctx, req.Context)
        }
    }
    return nil, sidecar.ErrMethodNotFound
}
```

接口边界约束：
- `central.review_trade` / `central.data_alert` / `central.account_error` / `central.macro_event` 是公开 tool，供其他 brain 通过 `specialist.call_tool` 调用
- `daily_review_run` / `update_config` / `health_check` 是内部 instruction，只由 Central 自身调度或人工触发
- Central Brain 不直接与其他 sidecar 建链，所有跨脑调用仍经 Kernel 路由
