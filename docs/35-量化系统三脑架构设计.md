### 1.1 设计原则

| 原则 | 说明 |
|------|------|
| **数据源插件化** | 所有数据源实现统一 `DataProvider` 接口，配置驱动热插拔 |
| **数据零拷贝** | 高频数据走共享内存 ring buffer，不经过 Kernel 中转 |
| **关键路径无 LLM** | tick→策略→下单全链路不调 LLM，延迟 < 5ms |
| **多账号隔离** | 每个账号独立风控、独立仓位、独立损益核算 |
| **多账号聚合** | 跨账号全局风控 + 中央大脑统一视图 |
| **故障隔离** | 三个大脑独立进程，任一崩溃不影响其他 |
| **双通道通信** | 快路径（共享内存）+ 慢路径（JSON-RPC） |
| **数据质量守门** | 所有数据经过校验层，异常数据不进入策略 |
| **全链路可追溯** | 每笔交易决策完整记录，支持回放审计 |
| **品种标准化** | 统一品种 ID，屏蔽各交易所命名差异 |

### 1.2 延迟预算（高频路径）

```
OKX WebSocket 消息到达
    │
    ├─ [0.1ms] Frame 解析 + 反序列化
    ├─ [0.1ms] 数据质量校验
    ├─ [0.2ms] 数据清洗 + K线聚合
    ├─ [0.3ms] 特征向量计算
    ├─ [0.1ms] 写入 ring buffer ──── Data Brain 完成
    │
    ├─ [0.1ms] Quant Brain 读取 ring buffer
    ├─ [0.5ms] 4 策略并行计算
    ├─ [0.2ms] 信号聚合 + 阈值判断
    ├─ [0.1ms] 全局风控检查
    ├─ [0.2ms] per-account 风控检查
    ├─ [0.1ms] 仓位计算
    ├─ [0.5ms] 多账号并行下单 ──── Quant Brain 完成
    │
    └─ 总计: < 2.5ms（P99 < 5ms）

注: 真正的瓶颈在服务器到 OKX 的网络延迟（50-200ms），
    内部处理延迟相比之下可忽略。
```

### 1.3 单机资源估算

本系统设计为**单机部署**，资源占用极低：

| 组件 | 内存 | CPU |
|------|------|-----|
| Data Brain 进程 | ~50 MB | < 5% |
| Ring Buffer（3 品种 × 1.75 MiB） | ~5 MB | — |
| Quant Brain 进程 | ~30 MB | < 5% |
| Central Brain 进程（大部分时间空闲） | ~20 MB | < 1% |
| Kernel 主进程 | ~30 MB | < 1% |
| PostgreSQL 连接池 | ~50 MB | < 3% |
| **总计** | **~210 MB** | **< 15%** |

- 网络带宽: < 100 KB/s（WebSocket 行情 + REST 补全）
- 磁盘写入: < 10 MB/天（K线 + 向量 + 交易记录）
- **最低要求: 2 核 4G VPS 即可运行**

## 二、数据大脑（Data Brain）完整方案

### 2.1 核心设计：DataProvider 插件架构

数据大脑不是 OKX 客户端，它是一个**统一数据中枢**。所有数据源通过 `DataProvider` 接口接入。

```go
// ========================
// 数据源插件核心抽象
// ========================

// DataCategory 数据分类 — 决定存储策略和消费方式
type DataCategory string

const (
    CategoryMarket      DataCategory = "market"      // 行情（K线、深度、逐笔）
    CategoryMacro       DataCategory = "macro"       // 宏观经济（非农、CPI、利率）
    CategoryNews        DataCategory = "news"        // 新闻/舆情
    CategoryOnChain     DataCategory = "on_chain"    // 链上数据（Gas、大额转账）
    CategoryEquity      DataCategory = "equity"      // 股票行情
    CategoryAlternative DataCategory = "alternative" // 另类数据（社交情绪、搜索趋势）
)

// DataProvider 统一数据源接口 — 所有数据源必须实现
type DataProvider interface {
    // 身份
    Name() string                          // "okx_swap", "reuters_news", "fred_macro"
    Category() DataCategory                // market, macro, news, ...
    Version() string                       // 语义版本号

    // 生命周期
    Start(ctx context.Context) error       // 启动采集（连接、订阅）
    Stop(ctx context.Context) error        // 优雅关闭
    Health() ProviderHealth                // 健康检查

    // 数据输出
    Subscribe(sink DataSink) error         // 注册下游消费者
}

// ReplayableProvider 支持历史回放的 Provider（用于回测）
type ReplayableProvider interface {
    DataProvider
    Replay(ctx context.Context, from, to time.Time, speed float64) error
}

// DataSink 数据消费端 — 由 Data Brain 核心实现
type DataSink interface {
    OnEvent(event DataEvent)
}

// DataEvent 统一数据事件 — 所有数据源的公共信封
type DataEvent struct {
    Provider   string            // 来源 provider 名称
    Category   DataCategory      // 数据分类
    Topic      string            // 细分主题: "candle.1m.BTC-USDT-SWAP", "nfp.actual"
    Symbol     string            // 标准化品种 ID（经 InstrumentNormalizer 转换）
    Timestamp  int64             // 数据产生时间（毫秒）
    LocalTS    int64             // 本地接收时间（纳秒）
    Priority   EventPriority     // 优先级（决定走快路径还是慢路径）
    Payload    any               // 具体数据（强类型，按 Category 向下转型）
}

// EventPriority 事件优先级
type EventPriority uint8

const (
    PriorityRealtime EventPriority = 1    // 实时: tick/深度 → 走共享内存
    PriorityNearRT   EventPriority = 2    // 准实时: 1分钟K线、资金费率 → 走内存 channel
    PriorityBatch    EventPriority = 3    // 批量: 新闻、宏观数据 → 走 JSON-RPC
)

// ProviderHealth 健康状态
type ProviderHealth struct {
    Status     string            // "healthy", "degraded", "disconnected"
    Latency    time.Duration     // 最近一条数据的延迟
    LastEvent  time.Time         // 最后一次收到数据
    ErrorCount int64             // 累计错误数
    Detail     string            // 人类可读描述
}
```

### 2.2 品种标准化层（Instrument Normalizer）

不同交易所对同一品种的命名不同，Data Brain 统一为标准化 ID：

```go
// InstrumentNormalizer 品种命名标准化
// 所有 Provider 的原始品种 ID 经过此层转换为统一格式
type InstrumentNormalizer struct {
    mappings map[string]map[string]string // exchange → rawID → stdID
}

// 标准格式: {BASE}-{QUOTE}-{TYPE}
// 示例:
//   OKX    "BTC-USDT-SWAP"   → "BTC-USDT-SWAP"   (恰好一致)
//   Binance "BTCUSDT"         → "BTC-USDT-PERP"
//   Bybit   "BTCUSDT"         → "BTC-USDT-PERP"
//   A股     "600519.SH"       → "600519-CNY-STOCK"

func (n *InstrumentNormalizer) Normalize(exchange, rawID string) string {
    if m, ok := n.mappings[exchange]; ok {
        if stdID, ok := m[rawID]; ok {
            return stdID
        }
    }
    return rawID // 未注册的品种保留原始 ID
}

// Denormalize 反向转换 — Executor 下单时需要
func (n *InstrumentNormalizer) Denormalize(exchange, stdID string) string {
    // 反向查找
}
```

### 2.5 回测 Provider（ReplayProvider）

```go
// ReplayProvider 从 PostgreSQL 读取历史数据回放
// 实现 ReplayableProvider 接口
type ReplayProvider struct {
    name      string
    store     *storage.Repositories
    speed     float64              // 1.0 = 实时速度, 0 = 尽速回放
    eventSink DataSink
}

func (p *ReplayProvider) Category() DataCategory { return CategoryMarket }

// Replay 按时间戳顺序回放历史数据
// Quant Brain 不需要任何修改 — 它只消费 ring buffer / MarketView
func (p *ReplayProvider) Replay(ctx context.Context, from, to time.Time, speed float64) error {
    candles, err := p.store.Candles().QueryRange(ctx, from, to)
    if err != nil {
        return err
    }

    var prevTS int64
    for _, c := range candles {
        // 模拟时间间隔
        if speed > 0 && prevTS > 0 {
            delay := time.Duration(float64(c.Timestamp-prevTS)*1e6) / time.Duration(speed)
            time.Sleep(delay)
        }
        prevTS = c.Timestamp

        p.eventSink.OnEvent(DataEvent{
            Provider:  p.name,
            Category:  CategoryMarket,
            Symbol:    c.InstID,
            Topic:     fmt.Sprintf("candle.%s.%s", c.Bar, c.InstID),
            Timestamp: c.Timestamp,
            LocalTS:   time.Now().UnixNano(),
            Priority:  PriorityNearRT,
            Payload:   c,
        })
    }
    return nil
}
```

### 2.7 数据质量校验层（DataValidator）

```go
// DataValidator 数据质量守门员
// 位于 InstrumentNormalizer 之后、EventRouter 之前
type DataValidator struct {
    lastPrice    map[string]float64    // symbol → 最后已知价格
    lastTS       map[string]int64      // symbol → 最后时间戳
    gapDetector  *GapDetector          // 缺失检测
    alertSink    func(alert ValidatorAlert) // 异常告警回调 → Central Brain
}

type ValidatorAlert struct {
    Level   string    // "warning", "critical"
    Type    string    // "price_spike", "gap", "stale", "inconsistent"
    Symbol  string
    Detail  string
    EventTS time.Time
}

// Validate 校验单个事件，返回 true = 通过，false = 丢弃
func (v *DataValidator) Validate(event *DataEvent) bool {
    switch event.Category {
    case CategoryMarket:
        return v.validateMarket(event)
    default:
        return true // 非行情数据暂不校验
    }
}

func (v *DataValidator) validateMarket(event *DataEvent) bool {
    // 1. 时间戳检查 — 不接受未来数据或过期数据
    now := time.Now().UnixMilli()
    if event.Timestamp > now+5000 { // 超前 5s 以上
        v.alert("warning", "future_timestamp", event.Symbol,
            fmt.Sprintf("ts=%d, now=%d", event.Timestamp, now))
        return false
    }
    if event.Timestamp < now-300000 { // 滞后 5 分钟以上
        v.alert("warning", "stale_data", event.Symbol,
            fmt.Sprintf("ts=%d, lag=%ds", event.Timestamp, (now-event.Timestamp)/1000))
        return false
    }

    // 2. 价格突变检测（仅对有价格的事件）
    if price := extractPrice(event); price > 0 {
        if last, ok := v.lastPrice[event.Symbol]; ok && last > 0 {
            change := math.Abs(price-last) / last
            if change > 0.10 { // 单 tick 价格变动 >10%
                v.alert("critical", "price_spike", event.Symbol,
                    fmt.Sprintf("%.2f → %.2f (%.1f%%)", last, price, change*100))
                return false
            }
        }
        v.lastPrice[event.Symbol] = price
    }

    // 3. 重复时间戳检测
    if last, ok := v.lastTS[event.Symbol]; ok && event.Timestamp <= last {
        // 允许同一毫秒的不同事件类型，但同 topic 重复则丢弃
        return event.Timestamp < last // 严格旧数据丢弃
    }
    v.lastTS[event.Symbol] = event.Timestamp

    // 4. K线缺失 gap 检测
    v.gapDetector.Observe(event)

    return true
}
```

### 2.8 Event Router — 按优先级分流

```go
type EventRouter struct {
    realtimeCh chan DataEvent       // 无缓冲，实时处理
    nearRTCh   chan DataEvent       // 缓冲 1024
    batchCh    chan DataEvent       // 缓冲 4096
}

func (r *EventRouter) OnEvent(event DataEvent) {
    switch event.Priority {
    case PriorityRealtime:
        select {
        case r.realtimeCh <- event:
        default:
            // tick 数据丢一个没关系，metric: realtime_drop_count++
        }
    case PriorityNearRT:
        r.nearRTCh <- event
    case PriorityBatch:
        r.batchCh <- event
    }
}
```

```go
    // 对齐填充
    _padding           [72]byte
    // 总大小: 1792 字节
}

// SharedRingBuffer — Data Brain 写，Quant Brain 读
type SharedRingBuffer struct {
    header *RingHeader
    slots  []MarketSnapshot          // 1024 个槽位
    fd     int
    path   string                    // /dev/shm/brain-data-{instrument}
}

type RingHeader struct {
    WriteSeq   uint64               // atomic: 写入者最新序号
    ReadSeq    uint64               // atomic: 读取者已读序号
    SlotCount  uint32               // 1024
    SlotSize   uint32               // 1792
    CreatedAt  int64
    _reserved  [32]byte
}

// 每品种 1024 × 1792 ≈ 1.75 MiB，3 品种总计 ≈ 5.25 MiB

// Write 由 Data Brain 调用
func (rb *SharedRingBuffer) Write(snap *MarketSnapshot) {
    seq := atomic.AddUint64(&rb.header.WriteSeq, 1)
    snap.SeqNum = seq
    idx := seq % uint64(rb.header.SlotCount)
    rb.slots[idx] = *snap
}

// ReadLatest 由 Quant Brain 调用
func (rb *SharedRingBuffer) ReadLatest() (*MarketSnapshot, bool) {
    writeSeq := atomic.LoadUint64(&rb.header.WriteSeq)
    readSeq := atomic.LoadUint64(&rb.header.ReadSeq)
    if writeSeq <= readSeq {
        return nil, false
    }
    idx := writeSeq % uint64(rb.header.SlotCount)
    snap := rb.slots[idx]
    atomic.StoreUint64(&rb.header.ReadSeq, writeSeq)
    return &snap, true
}
```

### 3.2 全局风控层（GlobalRiskGuard）

```go
// GlobalRiskGuard 跨账号全局风控
// 位于 Aggregator 之后、AccountRouter 之前
// 未来如果复杂度增长，可以独立为 Risk Brain sidecar
type GlobalRiskGuard struct {
    accounts     map[string]*AccountState
    config       GlobalRiskConfig
}

type GlobalRiskConfig struct {
    MaxGlobalExposurePct  float64    // 所有账号总敞口 / 总权益，默认 50%
    MaxGlobalSameDirection float64   // 同方向总敞口上限，默认 30%
    MaxGlobalDailyLoss    float64    // 所有账号累计日亏损上限，默认 5%
    MaxSymbolExposure     float64    // 单品种跨账号总敞口上限，默认 15%
}

// Evaluate 在信号分发到各账号之前做全局检查
func (g *GlobalRiskGuard) Evaluate(signal *strategy.AggregatedSignal) *risk.Decision {
    // 1. 计算全局总敞口
    totalEquity := 0.0
    totalExposure := 0.0
    symbolExposure := 0.0  // 该品种跨所有账号的敞口
    sameDirectionExp := 0.0
    dailyLossTotal := 0.0

    for _, acct := range g.accounts {
        if acct.Status != "active" {
            continue
        }
        eq := acct.Memory.Equity()
        totalEquity += eq

        for _, pos := range acct.Memory.Positions() {
            notional := math.Abs(pos.Notional)
            totalExposure += notional

            if pos.Symbol == signal.Symbol {
                symbolExposure += notional
            }
            if string(pos.Direction) == string(signal.Direction) {
                sameDirectionExp += notional
            }
        }

        dailyLossTotal += acct.DailyPnL
    }

    if totalEquity == 0 {
        return &risk.Decision{Allowed: true}
    }

    // 2. 全局敞口检查
    if totalExposure/totalEquity > g.config.MaxGlobalExposurePct/100 {
        return &risk.Decision{
            Allowed: false,
            Layer:   "global",
            Reason:  fmt.Sprintf("global exposure %.1f%% > limit %.1f%%",
                totalExposure/totalEquity*100, g.config.MaxGlobalExposurePct),
        }
    }

    // 3. 单品种跨账号敞口
    if symbolExposure/totalEquity > g.config.MaxSymbolExposure/100 {
        return &risk.Decision{
            Allowed: false,
            Layer:   "global",
            Reason:  fmt.Sprintf("%s global exposure %.1f%% > limit",
                signal.Symbol, symbolExposure/totalEquity*100),
        }
    }

    // 4. 同方向敞口
    if sameDirectionExp/totalEquity > g.config.MaxGlobalSameDirection/100 {
        return &risk.Decision{
            Allowed: false,
            Layer:   "global",
            Reason:  "global same-direction exposure exceeded",
        }
    }

    // 5. 全局日亏损
    if dailyLossTotal/totalEquity < -g.config.MaxGlobalDailyLoss/100 {
        return &risk.Decision{
            Allowed: false,
            Layer:   "global",
            Reason:  fmt.Sprintf("global daily loss %.1f%%", dailyLossTotal/totalEquity*100),
        }
    }

    return &risk.Decision{Allowed: true}
}
```

### 3.6 Tick Loop（完整版）

```go
func (q *QuantBrain) tick(ctx context.Context) {
    for instID, ring := range q.rings {
        snap, ok := ring.ReadLatest()
        if !ok {
            continue
        }

        // 1. 适配为 MarketView
        view := q.adaptSnapshot(instID, snap)

        // 2. 策略池计算（全局共享，只算一次）
        signals := q.pool.Compute(view)

        // 3. 信号聚合
        agg := q.aggregator.Aggregate(signals, q.buildReviewContext())
        if agg.Direction == strategy.DirectionFlat || agg.Direction == strategy.DirectionHold {
            continue
        }

        // 4. ★ 开始构建审计追踪
        trace := &SignalTrace{
            TraceID:    generateTraceID(),
            Timestamp:  time.Now(),
            Symbol:     instID,
            SnapshotSeq: snap.SeqNum,
            Price:      snap.CurrentPrice,
            Features:   snap.FeatureVector,
            Signals:    signals,
            Aggregated: *agg,
        }

        // 5. ★ 全局跨账号风控
        globalDecision := q.globalGuard.Evaluate(agg)
        trace.GlobalRisk = *globalDecision
        if !globalDecision.Allowed {
            q.traceStore.Save(ctx, trace)
            continue
        }

        // 6. LLM 审查（如触发条件满足）
        if q.NeedsReview(agg) {
            review := q.RequestReview(ctx, agg, snap)
            if !review.Approved {
                q.traceStore.Save(ctx, trace)
                continue
            }
        }

        // 7. 信号广播到所有账号 — 各账号独立风控+执行
        results := q.router.Dispatch(ctx, agg, snap)
        trace.AccountResults = results

        // 8. ★ 保存完整审计追踪
        q.traceStore.Save(ctx, trace)
    }

    // 9. 每个账号独立处理价格 tick（止损/止盈）
    for _, acct := range q.router.accounts {
        q.processAccountOrders(ctx, acct)
    }
}
```

### 3.7 Account Router — 信号广播

```go
type AccountRouter struct {
    accounts map[string]*AccountState
    normalizer *InstrumentNormalizer  // 下单时反向转换品种 ID
}

func (r *AccountRouter) Dispatch(ctx context.Context, signal *strategy.AggregatedSignal, snap *MarketSnapshot) []AccountTraceResult {
    results := make([]AccountTraceResult, 0, len(r.accounts))

    for _, acct := range r.accounts {
        tr := AccountTraceResult{AccountID: acct.Config.ID}
        start := time.Now()

        if !acct.Config.Enabled || acct.Status != "active" {
            continue
        }

        // 策略过滤
        if !acct.isStrategyAllowed(signal.DominantStrategy) {
            continue
        }

        // 品种过滤
        if !acct.isInstrumentAllowed(signal.Symbol) {
            continue
        }

        // per-account 风控
        decision := acct.Guard.Evaluate(acct.buildOrderRequest(signal, snap))
        tr.RiskResult = *decision
        if !decision.Allowed {
            results = append(results, tr)
            continue
        }

        // 仓位计算（应用 WeightFactor）
        intent := acct.buildOrderIntent(signal, snap)
        intent.Quantity = scaleQuantity(intent.Quantity, acct.Config.WeightFactor)

        // ★ 下单时反向转换品种 ID
        intent.Symbol = r.normalizer.Denormalize(acct.Config.Exchange, signal.Symbol)

        tr.Intent = &intent

        // 执行
        result, err := acct.Executor.Execute(ctx, intent)
        if err != nil {
            acct.Status = "error"
            acct.LastError = err.Error()
            tr.Latency = time.Since(start)
            results = append(results, tr)
            continue
        }

        acct.Memory.Update(result)
        tr.ExecResult = &result
        tr.Latency = time.Since(start)
        results = append(results, tr)
    }

    return results
}
```

## 四、三脑低延迟通信方案

### 4.1 双通道架构

```
┌──────────────────────────────────────────────────────────────────┐
│                       通信拓扑（单机部署）                         │
│                                                                  │
│                    Central Brain                                 │
│                   ┌────────────┐                                 │
│            JSON-RPC│  Kernel   │JSON-RPC                         │
│           (stdio)  │           │(stdio)                          │
│            ┌──────┘            └──────┐                          │
│            │                          │                          │
│     Data Brain                  Quant Brain                     │
│     ┌──────────┐                ┌──────────┐                    │
│     │          ├───────────────►│          │                    │
│     │          │  /dev/shm      │          │                    │
│     │          │  Ring Buffer   │          │                    │
│     └──────────┘  (< 0.1ms)    └──────────┘                    │
│                                                                  │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━                │
│  快路径: mmap ring buffer  |  慢路径: JSON-RPC via Kernel        │
│  延迟 < 0.1ms             |  延迟 1-5ms                         │
│  无序列化                  |  完整审计                           │
│  99% 流量（tick）          |  1% 流量（控制信号）                 │
│                            |                                    │
│  单机优势: 共享内存 =      |  未来如需分布式，可替换为           │
│  零网络开销的最优解         |  Unix Domain Socket / TCP           │
└──────────────────────────────────────────────────────────────────┘
```

### 4.2 通信场景矩阵

| 场景 | 发起方 | 接收方 | 通道 | 延迟 |
|------|--------|--------|------|------|
| 实时行情/特征推送 | Data | Quant | 共享内存 | < 0.1ms |
| 历史K线查询 | Quant | Data | JSON-RPC | < 50ms |
| 向量相似度查询 | Quant | Data | JSON-RPC | < 100ms |
| LLM 审查请求 | Quant | Central | JSON-RPC | < 10s |
| 强制平仓/暂停 | Central | Quant | JSON-RPC | < 100ms |
| 策略权重调整 | Central | Quant | JSON-RPC | < 100ms |
| 数据源健康检查 | Central | Data | JSON-RPC | < 1s |
| 数据质量告警 | Data | Central | JSON-RPC | < 1s |
| 每日复盘 | Quant | Central | JSON-RPC | < 30s |
| 宏观数据预警 | Data | Quant | ring buffer ContextFlags | < 0.1ms |
| 多账号状态汇总 | Quant | Central | JSON-RPC | < 100ms |
| 信号审计查询 | Central | Quant | JSON-RPC | < 200ms |

### 4.3 启动时序

```
T+0s    Kernel 启动
        │
T+0.1s  ├── 启动 Data Brain
        │   ├── 初始化 InstrumentNormalizer
        │   ├── 初始化 DataValidator
        │   ├── 注册所有 DataProvider（当前: okx_swap）
        │   ├── 创建 /dev/shm ring buffers
        │   ├── 各 Provider.Start()
        │   │   ├── OKX WebSocket 连接
        │   │   └── REST 回填历史 K 线
        │   └── 报告 "data_ready"（含 shm 路径）─── ~3-5s
        │
T+5s    ├── 启动 Quant Brain
        │   ├── 打开 ring buffers（只读 mmap）
        │   ├── 初始化 GlobalRiskGuard
        │   ├── 初始化所有 AccountState
        │   │   ├── Account A: Risk Guard + OKX Executor + Private WS
        │   │   ├── Account B: Risk Guard + OKX Executor + Private WS
        │   │   └── Account C: Risk Guard + Paper Executor
        │   ├── 从 PostgreSQL 恢复各账号 MemoryState
        │   ├── 初始化 SignalTraceStore
        │   └── 报告 "quant_ready" ─── ~1s
        │
T+6s    ├── 启动 Central Brain
        │   └── 报告 "central_ready" ─── ~0.5s
        │
T+6.5s  └── 系统就绪，tick loop 开始
```

### 4.4 故障处理

```
Data Brain 崩溃:
  → Quant Brain 检测 ring buffer 停滞（WriteSeq 不增长）
  → 所有账号停止开新仓，保持止损/止盈监控
  → DataValidator 最后已知价格用于止损判断
  → Kernel 重启 Data Brain，自动恢复

Quant Brain 崩溃:
  → ★ 最大风险点：持仓暂时无人监控
  → Kernel 立即重启
  → 从 PostgreSQL 恢复各账号 MemoryState
  → Private WS 重连，获取最新持仓状态
  → 恢复止损/止盈监控

Central Brain 崩溃:
  → Data + Quant 不受影响（独立运行）
  → LLM 审查超时降级为自动决策
  → Kernel 重启 Central Brain

单个账号执行器故障:
  → 仅该账号标记为 "error"
  → 其他账号不受影响
  → Central Brain 收到告警
  → 可通过 quant.account_pause 暂停

数据质量异常:
  → DataValidator 拦截异常数据
  → 告警发送到 Central Brain
  → 如果是价格 spike → 该品种暂停交易直到数据恢复
  → 如果是 K 线 gap → 触发 REST 回填
```

## 九、关键设计决策备忘

| 决策 | 选择 | 理由 |
|------|------|------|
| 三脑 vs 四脑 | 三脑（Data/Quant/Central） | 当前规模不需要独立 Risk Brain，但 GlobalRiskGuard 模块化预留 |
| 共享内存 vs Socket | /dev/shm mmap | 单机部署，零序列化开销；ring.go 接口预留未来替换 |
| Private WS 归属 | Quant Brain | 包含账户凭据，属于执行层；Data Brain 只处理公开数据 |
