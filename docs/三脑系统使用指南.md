# 三脑系统使用指南

## 架构概览

三脑系统由三个独立大脑组成，协同运作：

```
┌─────────────┐      ┌─────────────┐      ┌─────────────┐
│  数据大脑    │ ───→ │  量化大脑    │ ←──→ │  中央大脑    │
│  Data Brain  │      │  Quant Brain │      │ Central Brain│
│  行情采集    │      │  策略+执行   │      │  LLM 复审    │
│  特征计算    │      │  风控+交易   │      │  日终分析    │
└─────────────┘      └─────────────┘      └─────────────┘
```

| 大脑 | 职责 | 配置文件位置 |
|------|------|-------------|
| 数据大脑 | OKX 行情采集、192 维特征向量、Ring Buffer | `~/.brain/data-brain.yaml` |
| 量化大脑 | 策略聚合、风控、仓位计算、交易执行 | `~/.brain/quant-brain.yaml` |
| 中央大脑 | LLM 交易复审、日终分析、数据告警 | `~/.brain/central-brain.yaml` |

## 前置条件

### 安装（从源码编译）

```bash
# 编译所有二进制到 GOPATH/bin（铁律：所有 Go 二进制编译到此处）
go build -o $GOPATH/bin/brain                ./cmd/brain/
go build -o $GOPATH/bin/brain-data-sidecar   ./brains/data/cmd/brain-data-sidecar/
go build -o $GOPATH/bin/brain-quant-sidecar  ./brains/quant/cmd/brain-quant-sidecar/
go build -o $GOPATH/bin/quant-brain          ./brains/quant/cmd/
```

### PostgreSQL（推荐）

```bash
# 创建数据库
createdb brain

# 设置连接串
export PG_URL="postgres://user:pass@localhost:5432/brain"
```

启用 PG 后自动获得：
- K 线/特征向量历史存储（数据大脑）
- 交易记录持久化（量化大脑）
- 信号追踪链审计（量化大脑）
- Paper 账户状态持久化（重启不丢数据）
- 崩溃恢复能力

## 快速开始（推荐方式：brain chat）

**最简单的启动方式**，一个命令启动全部三脑：

```bash
# 1. 配置文件放到 ~/.brain/ 目录下
cp quant-brain.yaml ~/.brain/quant-brain.yaml

# 2. 设置数据库（可选但推荐）
export PG_URL="postgres://brain:secret@localhost:5432/brain"

# 3. 启动！
brain chat
```

**brain chat 会自动**：
- 启动数据大脑 sidecar（采集行情、计算特征向量）
- 启动量化大脑 sidecar（策略评估、风控、交易执行）
- 自动发现 `~/.brain/quant-brain.yaml` 并注入 `QUANT_CONFIG` 环境变量
- 自动发现 `~/.brain/data-brain.yaml` 并注入 `DATA_CONFIG` 环境变量
- 提供 LLM 交互界面，可直接调用所有大脑工具

**无需手动设置任何环境变量**（PG_URL 除外），kernel 会自动完成配置注入。

### 配置文件自动发现规则

kernel 启动 sidecar 时，按以下映射自动注入环境变量：

| Sidecar | 环境变量 | 自动搜索路径 |
|---------|---------|-------------|
| data-brain | `DATA_CONFIG` | `~/.brain/data-brain.yaml` |
| quant-brain | `QUANT_CONFIG` | `~/.brain/quant-brain.yaml` |
| central-brain | `CENTRAL_CONFIG` | `~/.brain/central-brain.yaml` |

- 如果文件存在且对应环境变量未手动设置 → **自动注入**
- 如果文件不存在 → sidecar 使用内置默认配置（paper 模式）
- 如果用户手动设置了环境变量 → 使用用户指定的路径

## 量化配置文件详解

配置文件位置：`~/.brain/quant-brain.yaml`

完整参考手册见配置文件内嵌注释。以下是核心概念说明。

### 最简配置（一行 auto_risk）

```yaml
brain:
  cycle_interval: 1s
  default_timeframe: "1m"

accounts:
  - id: paper-main
    exchange: paper
    initial_equity: 10000

units:
  - id: unit-active
    account_id: paper-main
    timeframe: "1m"
    max_leverage: 10
    enabled: true

strategy:
  weights:
    TrendFollower: 0.30
    MeanReversion: 0.25
    BreakoutMomentum: 0.25
    OrderFlow: 0.20
  long_threshold: 0.45
  short_threshold: 0.45
  min_active_strategies: 2

# 一行搞定风控！系统根据账户资金自动计算所有参数
auto_risk:
  enabled: true
  level: aggressive
```

### auto_risk 自动风控（重点功能）

启用 `auto_risk` 后，**不再需要手动配置** `risk.guard`、`risk.position_sizer`、`global_risk` 的每个数值。系统根据账户实际权益和风险等级自动推导。

| 风险等级 | 适用场景 | 单笔上限 | 总敞口 | 最大并发 | Kelly 缩放 |
|---------|---------|---------|--------|---------|-----------|
| `conservative` | 实盘、大资金 | 2% | 20% | 3 笔 | 0.15 |
| `moderate` | 中等资金 | 5% | 40% | 8 笔 | 0.25 |
| `aggressive` | 纸盘数据积累 | 8% | 80% | 15 笔 | 0.35 |

**不同资金量的自动适配效果**：

| 本金 | 最小单笔 | 最大单笔 | 最大并发 | 说明 |
|------|---------|---------|---------|------|
| 50 USDT | 5 U (10%) | 7.5 U (15%) | 2 笔 | 微型账户，仅开 2 笔 |
| 1,000 USDT | 5 U (0.5%) | 50 U (5%) | 8 笔 | 小账户 |
| 10,000 USDT | 20 U (0.2%) | 800 U (8%) | 15 笔 | 标准纸盘 |
| 100,000 USDT | 200 U (0.2%) | 3000 U (3%) | 4 笔 | 大资金保守 |

**手动值作为天花板**：`auto_risk` 计算的值不会超过 `risk` 段手动配置的非零值。两者共存时，取较小值。

### 策略配置

4 个策略加权聚合，权重之和建议为 1.0：

| 策略 | 默认权重 | 原理 |
|------|----------|------|
| TrendFollower | 0.30 | 趋势跟踪（EMA 交叉 + ADX + MACD） |
| MeanReversion | 0.25 | 均值回归（布林带 + RSI 超买超卖） |
| BreakoutMomentum | 0.25 | 突破动量（区间突破 + 成交量放大） |
| OrderFlow | 0.20 | 订单流（买卖压力 + 订单簿失衡） |

关键参数：
- `long_threshold` / `short_threshold`：聚合信心度阈值（0.45 = 需 45% 以上信心）
- `min_active_strategies`：最少 N 个策略出方向信号才交易（2 = 防止单策略噪音）
- `dominance_factor`：主导策略放大系数（1.5 = 最强策略额外加权）

**时间周期自适应**：1m/5m 周期自动降低阈值（×0.80），因为短周期信号弱且相关性低。

### 信号反转平仓

```yaml
signal_exit:
  enabled: true
  min_confidence: 0.5
  require_multi_strategy: 2
```

持有多仓时检测到 ≥2 个策略一致做空信号（信心度 ≥ 50%）→ 自动平掉多仓。比固定止损更智能。

## 独立启动（不通过 brain chat）

### 纸盘快速测试

```bash
quant-brain -paper -equity 10000
```

### 配置文件启动

```bash
# 基础
quant-brain -config ~/.brain/quant-brain.yaml

# 带 PostgreSQL
quant-brain -config ~/.brain/quant-brain.yaml -pg "$PG_URL"
```

### 生产部署（三进程分开启动）

```bash
export PG_URL="postgres://brain:secret@db:5432/brain"
export LLM_API_KEY="sk-xxx"

# 终端 1：数据大脑
brain-data -pg "$PG_URL" -backfill -backfill-days 90

# 终端 2：量化大脑
quant-brain -config ~/.brain/quant-brain.yaml -pg "$PG_URL"

# 终端 3：中央大脑（可选，提供 LLM 复审）
brain-central
```

## 风控体系

### 三层风控检查

每笔交易必须依次通过三层检查：

```
信号 → [第一层] 单笔检查 → [第二层] 组合检查 → [第三层] 熔断检查 → 执行
        杠杆、止损距离     并发数、总敞口      日亏损阈值
```

### 自适应风控（AdaptiveGuard）

风控参数根据市场波动率自动收紧：

| 波动率分位 | MaxPosition | MaxExposure | MaxConcurrent |
|-----------|-------------|-------------|---------------|
| 正常 (<75%) | 100% | 100% | 100% |
| 高波动 (75-90%) | ×0.6 | ×0.7 | ×0.6 |
| 极端波动 (>90%) | ×0.4 | ×0.5 | ×0.4 |

### 跨账户全局风控（GlobalRiskGuard）

多账户场景下的聚合风控：

| 阈值 | 默认值 | 说明 |
|------|--------|------|
| 总敞口上限 | 50% | 所有账号总敞口 / 总权益 |
| 同方向上限 | 30% | 同方向总敞口 / 总权益 |
| 日亏损上限 | 5% | 所有账号累计日亏损 |
| 单品种上限 | 15% | 单品种跨账号总敞口 |

（启用 auto_risk 时，以上值根据风险等级自动调整）

## LLM 复审

### 触发条件（满足任一）

- 持仓数 ≥ 3
- 最大单仓占比 > 5%
- 当日亏损 > 3%

### 复审流程

```
量化大脑 → specialist.call_tool → 中央大脑 → LLM → 批准/拒绝 + 仓位系数
```

LLM 超时或不可用时自动放行（size_factor=1.0）。

### 支持的 LLM 服务

| 服务商 | LLM_BASE_URL | LLM_MODEL | 备注 |
|--------|-------------|-----------|------|
| DeepSeek | `https://api.deepseek.com/v1` | `deepseek-chat` | 默认推荐 |
| Claude | `https://api.anthropic.com/v1` | `claude-sonnet-4-20250514` | Anthropic |
| 腾讯混元 | `https://api.hunyuan.cloud.tencent.com/v1` | `hunyuan-pro` | |
| 自定义 | 任何 OpenAI 兼容 API | 自定义 | 需支持 chat/completions |

## 崩溃恢复

启用 PostgreSQL 后自动支持：
1. 从 PGStore 加载历史交易记录
2. 与交易所核对当前持仓
3. 10 tick 预热期（策略重建状态，不开新仓）
4. Paper 账户余额/持仓从 PG 恢复（重启不归零）

## 常见问题

### Q: 策略显示 "feature vector not ready"

**原因**：量化大脑尚未从数据大脑获取到特征向量数据。

**解决**：
1. 确认数据大脑已启动（`brain chat` 模式下自动启动）
2. 等待 2-5 秒（remoteBuf 有启动时退避重试机制）
3. 检查日志中是否有 `remote snapshot fetch failed` 错误

### Q: 换了资金量需要改配置吗？

**不需要。** 启用 `auto_risk` 后，系统根据账户实际权益自动计算所有风控参数。
只需设置风险等级（conservative/moderate/aggressive）。

### Q: 纸盘交易太少/资金闲置

调整 auto_risk level 为 `aggressive`，或降低 `strategy.min_active_strategies` 为 1。

### Q: 如何查看实时交易状态？

```bash
brain chat
# 然后输入：
# "查看全局持仓"
# "查看今日盈亏"
# "查看策略权重"
# "查看风控状态"
```
