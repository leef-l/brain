# 三脑系统使用指南

## 架构概览

三脑系统由三个独立大脑组成，协同运作：

```
┌─────────────┐      ┌─────────────┐      ┌─────────────┐
│  数据大脑    │ ───→ │  量化大脑    │ ←──→ │  中央大脑    │
│  Data Brain  │      │  Quant Brain │      │ Central Brain│
│  行情采集    │      │  策略+执行   │      │  LLM 复审    │
│  特征计算    │      │  风控+交易   │      │  日终分析    │
└─────────────┘      └─────────────┘      └─────────────┘
```

| 大脑 | 职责 | 配置文件 |
|------|------|----------|
| 数据大脑 | OKX 行情采集、192 维特征向量、Ring Buffer | `brains/data/config.example.yaml` |
| 量化大脑 | 策略聚合、风控、仓位计算、交易执行 | `brains/quant/config.example.yaml` |
| 中央大脑 | LLM 交易复审、日终分析、数据告警 | `central/config.example.yaml` |

---

## 前置条件

### 安装

```bash
# 方式 1：pnpm 全局安装（推荐，10 个二进制开箱即用）
pnpm add -g @leef-l/brain-cli

# 方式 2：从源码编译
go build -o bin/brain              ./cmd/brain/
go build -o bin/brain-data         ./brains/data/cmd/
go build -o bin/brain-data-sidecar ./brains/data/cmd/sidecar/
go build -o bin/brain-quant        ./brains/quant/cmd/
go build -o bin/brain-quant-sidecar ./brains/quant/cmd/sidecar/
go build -o bin/brain-central      ./central/cmd/
```

### PostgreSQL（可选但推荐）

```bash
# 创建数据库
createdb brain

# 连接串格式
export PG_URL="postgres://user:pass@localhost:5432/brain"
```

启用 PG 后自动获得：
- K 线/特征向量历史存储（数据大脑）
- 交易记录持久化（量化大脑）
- 信号追踪链审计（量化大脑）
- 崩溃恢复能力（量化大脑）

## 快速开始（纸盘模式）

最简单的启动方式，无需 API Key、数据库或配置文件：

```bash
# 终端 1：量化大脑（内置纸盘交易所 + 简易数据源）
quant-brain -paper -equity 10000
```

纸盘模式自动：
- 创建 paper 交易所模拟器
- 使用内存存储（重启丢失）
- 无需 LLM 复审

## 生产部署

### 第一步：启动数据大脑

```bash
# 基础启动（无持久化）
brain-data

# 指定合约 + 历史回填
brain-data -instruments BTC-USDT-SWAP,ETH-USDT-SWAP -backfill -backfill-days 30

# 完整模式（持久化 + 回填）
brain-data -pg "$PG_URL" -backfill -backfill-days 90
```

数据大脑会：
1. 自动发现高成交量合约（或使用指定列表）
2. 建立 WebSocket 连接采集实时行情
3. 计算 192 维特征向量（1 秒刷新）
4. 通过 Ring Buffer 供量化大脑读取

### 第二步：启动量化大脑

```bash
# 配置文件启动
quant-brain -config config.yaml

# 带 PostgreSQL 持久化
quant-brain -config config.yaml -pg "$PG_URL"
```

配置文件格式为 YAML（也支持 JSON），详见 `brains/quant/config.example.yaml`。

量化大脑核心流程：
```
特征向量 → 4 策略评分 → 加权聚合 → 全局风控 → [LLM 复审] → 仓位计算 → 执行
```

### 第三步：启动中央大脑（可选）

中央大脑提供 LLM 智能复审，需要 LLM API 密钥：

```bash
# DeepSeek V3.2（默认，推荐）
export LLM_API_KEY="sk-xxx"
brain-central

# 自定义 LLM 服务
LLM_API_KEY="sk-xxx" \
LLM_BASE_URL="https://api.example.com/v1" \
LLM_MODEL="model-name" \
brain-central
```

不启动中央大脑时，量化大脑的 LLM 复审请求会自动放行（NullReviewer）。

## 环境变量一览

| 变量 | 用途 | 默认值 | 使用者 |
|------|------|--------|--------|
| `PG_URL` | PostgreSQL 连接串 | 无（内存模式） | 数据大脑、量化大脑 |
| `LLM_API_KEY` | LLM API 密钥 | 无（禁用 LLM） | 中央大脑 |
| `LLM_BASE_URL` | LLM API 地址 | `https://api.deepseek.com/v1` | 中央大脑 |
| `LLM_MODEL` | LLM 模型名称 | `deepseek-chat` | 中央大脑 |

## 配置文件格式

所有配置文件统一使用 **YAML 格式**（`.yaml` 或 `.yml`），代码同时兼容 JSON。

YAML 的优势：支持注释，便于文档化配置项。

### 量化大脑配置结构

```yaml
brain:
  cycle_interval: 5s          # 评估周期
  default_timeframe: "1H"     # 默认 K 线周期

accounts:                      # 交易账户列表
  - id: okx-main
    exchange: okx              # paper | okx
    api_key: "xxx"
    secret_key: "xxx"
    passphrase: "xxx"
    route:                     # 账户路由（可选）
      weight_factor: 1.0       # 仓位权重因子
      allowed_strategies: []   # 限制策略（空=全部）
      allowed_symbols: []      # 限制品种（空=全部）

units:                         # 交易单元列表
  - id: unit-btc
    account_id: okx-main
    symbols: [BTC-USDT-SWAP]
    timeframe: "1H"
    max_leverage: 10
    enabled: true
```

完整配置示例：`brains/quant/config.example.yaml`

## 模块说明

### 策略层（4 策略加权聚合）

| 策略 | 默认权重 | 说明 |
|------|----------|------|
| TrendFollower | 0.30 | 趋势跟踪（EMA 交叉 + ADX） |
| MeanReversion | 0.25 | 均值回归（布林带 + RSI） |
| BreakoutMomentum | 0.25 | 突破动量（区间突破 + 成交量） |
| OrderFlow | 0.20 | 订单流（买卖压力 + 订单簿失衡） |

聚合规则：
- 做多/做空分数 ≥ 0.45
- 主方向分数 ≥ 反方向 × 1.5
- 历史胜率过滤（做多胜率 < 35% 拒绝，做空胜率 > 65% 拒绝）

### 风控层

**单账户守卫（Guard）**：杠杆、止损距离、仓位占比、日亏损熔断

**跨账户全局风控（GlobalRiskGuard）**：

| 阈值 | 默认值 | 说明 |
|------|--------|------|
| 总敞口上限 | 50% | 所有账号总敞口 / 总权益 |
| 同方向上限 | 30% | 同方向总敞口 / 总权益 |
| 日亏损上限 | 5% | 所有账号累计日亏损 |
| 单品种上限 | 15% | 单品种跨账号总敞口 |

### LLM 复审

触发条件（满足任一）：
- 持仓数 ≥ 3
- 最大单仓占比 > 5%
- 当日亏损 > 3%

复审流程：量化大脑 → 中央大脑 → LLM → 批准/拒绝 + 仓位系数

LLM 超时或不可用时自动放行（size_factor=1.0）。

### 崩溃恢复

启用 PostgreSQL 后自动支持：
1. 从 PGStore 加载历史交易记录
2. 与交易所核对当前持仓
3. 10 tick 预热期（策略重建状态，不开新仓）

### 信号追踪链（SignalTrace）

完整记录每个决策链路：
```
特征快照 → 策略信号 → 聚合结果 → 全局风控 → 各账户执行结果 → 最终结果
```

启用 PG 后自动持久化到 `signal_traces` 表。

## 通过 brain CLI 使用（sidecar 模式）

v0.7.0 起，`brain chat` 和 `brain run` 可以**直接调用**数据大脑和量化大脑的全部工具，无需单独启动三个进程。

### 配置

在 `~/.brain/config.json` 中声明专精大脑 sidecar：

```json
{
  "brains": {
    "data": {
      "cmd": "brain-data-sidecar",
      "env": { "DATA_CONFIG": "/path/to/data-config.json" }
    },
    "quant": {
      "cmd": "brain-quant-sidecar",
      "env": { "QUANT_CONFIG": "/path/to/quant-config.yaml" }
    }
  }
}
```

### 使用

```bash
# 交互式 chat — LLM 可直接调用 quant.*/data.* 工具
brain chat

# 单次查询
brain run --prompt "查看所有账户余额和持仓"
brain run --prompt "查询 BTC-USDT-SWAP 的实时价格和特征向量"
brain run --prompt "查看今日各交易单元的盈亏"
brain run --prompt "暂停所有交易"
```
