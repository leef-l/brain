# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| `0.6.x` | Yes |
| `0.5.x` | Yes |
| `< 0.5.0` | No |

## Reporting a Vulnerability

Do not open a public GitHub issue for security reports.

Please use one of these private channels:

- GitHub Security Advisory: <https://github.com/leef-l/brain/security/advisories/new>
- GitHub private maintainer contact: <https://github.com/leef-l>

When reporting, include:

- affected version or commit
- reproduction steps or proof of concept
- impact assessment
- any suggested mitigation or patch

## Response SLA

- Initial acknowledgement: within 72 hours
- Status update: at least once every 7 days while triaging
- Target remediation window: 30 days for high-severity issues when a fix is feasible

## Disclosure Policy

- Please allow time for triage and a coordinated fix before public disclosure.
- If the issue affects released binaries, the preferred remediation path is a new GitHub Release with updated checksums and artifact attestation.
