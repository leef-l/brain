## 一、定位与边界

### 数据大脑是什么

数据大脑是整个交易系统的**感知层**，类比人的"眼睛 + 视觉皮层"。

它的输出是一个**增强特征向量**——无论底层是规则计算还是 ML 推理，对量化大脑来说都是透明的。

### 职责

| 做 | 不做 |
|----|------|
| 采集原始行情（WebSocket / REST） | 交易决策 |
| 数据清洗与验证 | 信号生成 |
| 增量技术指标计算 | 仓位计算 |
| 192 维特征向量生成 | 风险控制 |
| **ML 推理**（市场分类、波动率预测、异常检测） | 模型训练（离线任务） |
| 通过 Ring Buffer 输出增强特征 | 用户交互 |

### 核心设计原则

**ML 是感知的一部分，不是决策的一部分。**

- 波动率预测 → 感知（"市场现在有多剧烈"）→ 数据大脑
- 市场状态分类 → 感知（"现在是趋势市还是震荡市"）→ 数据大脑
- 异常检测 → 感知（"当前行为是否异常"）→ 数据大脑
- 做多还是做空 → 决策 → 量化大脑
- 仓位多大 → 决策 → 量化大脑

### 无 ML 降级原则

**数据大脑在没有 ML 模型时必须完全可用。**

每个 ML 维度都有一个规则降级计算（Rule Fallback），保证特征向量始终有值：

```
特征输出 = ML模型可用 ? ML推理结果 : 规则降级计算
```

量化大脑不需要知道某个维度是 ML 算的还是规则算的——向量就是向量。

### 3.7 ML 增强特征 [176:192] — 核心新增

这 16 维是 ML 推理输出。**无 ML 时全部由规则降级计算填充。**

#### MarketRegime [176:180] — 市场状态分类

| 索引 | 维度 | ML 输出 | 规则降级 |
|------|------|---------|---------|
| 176 | P(趋势) | softmax 概率 | ADX>25 && EMA排列 ? 0.8 : 0.2 |
| 177 | P(震荡) | softmax 概率 | ADX<20 && BB窄幅 ? 0.8 : 0.2 |
| 178 | P(突破) | softmax 概率 | 价格触及BB边界 && 放量 ? 0.7 : 0.1 |
| 179 | P(恐慌) | softmax 概率 | 跌幅>3ATR && 放量>3x ? 0.8 : 0.05 |

**约束：** 4 个概率之和 = 1.0（ML 输出天然满足，规则降级做归一化）。

**量化大脑使用方式：** 读取 [176:179]，选最大概率对应的状态，动态调整策略权重。

#### VolPredict [180:184] — 波动率预测

| 索引 | 维度 | ML 输出 | 规则降级 |
|------|------|---------|---------|
| 180 | 未来 1H 预测波动率 | GARCH/LSTM 输出 | 当前 ATR(14) / price (1H) |
| 181 | 未来 4H 预测波动率 | GARCH/LSTM 输出 | 当前 ATR(14) / price (4H) |
| 182 | 波动率百分位 (1H) | 100日历史百分位 | 当前vol / 历史vol中位数 |
| 183 | 波动率方向 | 预测vol - 当前vol | vol5 / vol20 - 1 |

**量化大脑使用方式：** 读取 [180:183]，高波动率时自动收紧仓位（AdaptiveGuard）。

#### AnomalyScore [184:188] — 异常检测

| 索引 | 维度 | ML 输出 | 规则降级 |
|------|------|---------|---------|
| 184 | 价格异常分 | Isolation Forest 分数 | abs(price_change) > 3×ATR ? 1 : 0 |
| 185 | 成交量异常分 | 模型输出 | vol > 5×avgVol ? 1 : 0 |
| 186 | 订单簿异常分 | 模型输出 | abs(imbalance)>0.8 && spread>3×avg ? 1:0 |
| 187 | 综合异常分 | 加权融合 | max(184, 185, 186) |

**量化大脑使用方式：** [187] > 0.7 时触发 ReviewContext.AnomalyDetected，交由 LLM Advisor 审查。

#### Reserved [188:192]

预留给未来扩展（链上数据、宏观指标、情绪指标等），默认为 0。

## 四、ML 推理框架

### 4.1 MLEngine 接口

```go
// MLEngine 是数据大脑内部的 ML 推理引擎。
// 它不是一个策略（不产生 long/short 信号），
// 它是特征增强器（把原始特征转化为更高级的感知）。
type MLEngine interface {
    // Predict 接收规则特征向量 [0:176]，输出 ML 增强维度 [176:192]。
    Predict(ctx context.Context, ruleFeatures []float64) (MLFeatures, error)

    // Ready 报告模型是否已加载就绪。
    // false 时 FeatureAssembler 自动使用规则降级。
    Ready() bool

    // Name 返回引擎名称（用于监控和日志）。
    Name() string
}

type MLFeatures struct {
    MarketRegime [4]float64  // [176:180]
    VolPredict   [4]float64  // [180:184]
    AnomalyScore [4]float64  // [184:188]
    Reserved     [4]float64  // [188:192]
}
```

### 4.2 实现方式（可选，按需接入）

| 实现 | 说明 | 适用场景 |
|------|------|---------|
| `NullMLEngine` | 永远返回 Ready()=false，触发规则降级 | 初期 / 无 ML |
| `ONNXMLEngine` | 本地加载 ONNX 模型文件推理 | 低延迟要求 |
| `HTTPMLEngine` | 调用外部 HTTP 推理服务 | Python 模型 |
| `SubprocessMLEngine` | 管理 Python 子进程通信 | 开发调试 |

### 4.3 NullMLEngine（默认实现）

```go
// NullMLEngine 是 MLEngine 的空实现。
// 永远返回 Ready()=false，让 FeatureAssembler 使用规则降级。
// 这是数据大脑的默认配置——无 ML 时系统完全可用。
type NullMLEngine struct{}

func (NullMLEngine) Predict(ctx context.Context, f []float64) (MLFeatures, error) {
    return MLFeatures{}, fmt.Errorf("NullMLEngine: no model loaded")
}
func (NullMLEngine) Ready() bool  { return false }
func (NullMLEngine) Name() string { return "null" }
```

### 4.4 规则降级实现

```go
// RuleFallback 为 ML 增强维度 [176:192] 提供纯规则计算的降级值。
// 当 MLEngine.Ready() == false 时，FeatureAssembler 调用此函数。
type RuleFallback struct{}

// Compute 从规则特征 [0:176] + 处理器状态计算降级的 ML 维度。
func (f *RuleFallback) Compute(
    ruleFeatures []float64,
    candles *CandleAggregator,
    orderbook *OrderBookTracker,
    tradeflow *TradeFlowTracker,
    instID string,
) MLFeatures {
    var ml MLFeatures

    // === MarketRegime [176:180] 规则降级 ===
    // 使用 1H 时间框架的指标判断
    w := candles.GetWindow(instID, "1H")
    if w != nil && w.ADX14.Ready() && w.EMA9.Ready() && w.BB20.Ready() {
        adx := w.ADX14.Value()
        emaAligned := w.EMA9.Value() > w.EMA21.Value() && w.EMA21.Value() > w.EMA55.Value() ||
                      w.EMA9.Value() < w.EMA21.Value() && w.EMA21.Value() < w.EMA55.Value()
        bbWidth := w.BBWidth()
        price := w.Current.Close
        atr := w.ATR14.Value()

        // 趋势概率
        if adx > 25 && emaAligned {
            ml.MarketRegime[0] = 0.7
        } else if adx > 20 {
            ml.MarketRegime[0] = 0.4
        } else {
            ml.MarketRegime[0] = 0.15
        }

        // 震荡概率
        if adx < 20 && bbWidth < atr*1.5 {
            ml.MarketRegime[1] = 0.7
        } else if adx < 25 {
            ml.MarketRegime[1] = 0.4
        } else {
            ml.MarketRegime[1] = 0.1
        }

        // 突破概率
        bbPos := w.BBPosition()
        volRatio := 1.0 // 默认
        if ruleFeatures[60] > 0 {
            volRatio = ruleFeatures[60] // Volume ratio
        }
        if (bbPos > 0.95 || bbPos < 0.05) && volRatio > 2.0 {
            ml.MarketRegime[2] = 0.7
        } else if bbPos > 0.85 || bbPos < 0.15 {
            ml.MarketRegime[2] = 0.3
        } else {
            ml.MarketRegime[2] = 0.05
        }

        // 恐慌概率
        priceMove := 0.0
        if atr > 0 {
            priceMove = math.Abs(w.PriceChangeRate(1)) * price / atr
        }
        if priceMove > 3 && volRatio > 3 {
            ml.MarketRegime[3] = 0.7
        } else if priceMove > 2 {
            ml.MarketRegime[3] = 0.2
        } else {
            ml.MarketRegime[3] = 0.05
        }

        // 归一化为概率分布（和 = 1.0）
        normalizeProb(ml.MarketRegime[:])
    } else {
        // 指标未就绪，均匀分布
        ml.MarketRegime = [4]float64{0.25, 0.25, 0.25, 0.25}
    }

    // === VolPredict [180:184] 规则降级 ===
    // 用当前 ATR 作为波动率预测的降级值
    if w != nil && w.ATR14.Ready() {
        price := w.Current.Close
        if price > 0 {
            ml.VolPredict[0] = w.ATR14.Value() / price       // 1H 预测 ≈ 当前 ATR
        }
        // 4H ATR
        if w4 := candles.GetWindow(instID, "4H"); w4 != nil && w4.ATR14.Ready() && w4.Current.Close > 0 {
            ml.VolPredict[1] = w4.ATR14.Value() / w4.Current.Close
        }
        // 波动率百分位：vol5/vol20 作为粗略百分位
        vol5 := w.Volatility(5)
        vol20 := w.Volatility(20)
        if vol20 > 0 {
            ml.VolPredict[2] = math.Min(vol5/vol20, 3.0) / 3.0 // 归一化到 [0, 1]
        } else {
            ml.VolPredict[2] = 0.5
        }
        // 波动率方向
        ml.VolPredict[3] = vol5/math.Max(vol20, 1e-10) - 1
    }

    // === AnomalyScore [184:188] 规则降级 ===
    if w != nil && w.ATR14.Ready() {
        price := w.Current.Close
        atr := w.ATR14.Value()
        // 价格异常
        if atr > 0 {
            priceAnomaly := math.Abs(w.PriceChangeRate(1)) * price / atr
            ml.AnomalyScore[0] = math.Min(priceAnomaly/3.0, 1.0) // > 3ATR → 1.0
        }
    }
    // 成交量异常
    if ruleFeatures[60] > 0 { // Volume ratio
        ml.AnomalyScore[1] = math.Min(ruleFeatures[60]/5.0, 1.0) // > 5x avg → 1.0
    }
    // 订单簿异常
    if ob := orderbook.Get(instID); ob != nil {
        obAnomaly := math.Abs(ob.Imbalance)
        if obAnomaly > 0.8 {
            ml.AnomalyScore[2] = obAnomaly
        }
    }
    // 综合异常
    ml.AnomalyScore[3] = math.Max(ml.AnomalyScore[0],
        math.Max(ml.AnomalyScore[1], ml.AnomalyScore[2]))

    return ml
}

func normalizeProb(p []float64) {
    sum := 0.0
    for _, v := range p { sum += v }
    if sum > 0 {
        for i := range p { p[i] /= sum }
    }
}
```

## 五、FeatureAssembler — 向量组装器

这是 Layer 3 的核心组件，负责将规则特征和 ML 特征合并成最终的 192 维向量。

```go
// FeatureAssembler 组装最终的 192 维特征向量。
// 它是 RuleFeatureEngine 和 MLFeatureEngine 的统一出口。
type FeatureAssembler struct {
    rule     *RuleFeatureEngine   // 计算 [0:176]
    ml       MLEngine             // 计算 [176:192]，可能为 NullMLEngine
    fallback *RuleFallback        // ML 不可用时的降级计算
    candles  *CandleAggregator    // 降级计算需要
    orderbook *OrderBookTracker
    tradeflow *TradeFlowTracker
}

// Compute 生成完整的 192 维向量。
func (a *FeatureAssembler) Compute(instID string) (FeatureOutput, error) {
    // 1. 规则特征 [0:176]
    ruleVec := a.rule.Compute(instID)

    // 2. ML 增强特征 [176:192]
    var mlFeatures MLFeatures
    var mlSource string

    if a.ml.Ready() {
        var err error
        mlFeatures, err = a.ml.Predict(context.Background(), ruleVec[:176])
        if err != nil {
            // ML 推理失败，降级
            mlFeatures = a.fallback.Compute(ruleVec, a.candles, a.orderbook, a.tradeflow, instID)
            mlSource = "fallback"
        } else {
            mlSource = a.ml.Name()
        }
    } else {
        // ML 未就绪，规则降级
        mlFeatures = a.fallback.Compute(ruleVec, a.candles, a.orderbook, a.tradeflow, instID)
        mlSource = "fallback"
    }

    // 3. 组装
    vec := ruleVec
    copy(vec[176:180], mlFeatures.MarketRegime[:])
    copy(vec[180:184], mlFeatures.VolPredict[:])
    copy(vec[184:188], mlFeatures.AnomalyScore[:])
    copy(vec[188:192], mlFeatures.Reserved[:])

    return FeatureOutput{
        Vector:   vec,
        MLSource: mlSource,
        MLReady:  a.ml.Ready(),
    }, nil
}

type FeatureOutput struct {
    Vector   [192]float64
    MLSource string  // "null", "fallback", "onnx_v1", "lstm_v2" 等
    MLReady  bool    // ML 模型是否在线
}
```

## 十、ML 平滑过渡路径

### Phase 0: 当前（无 ML）

```
MLEngine = NullMLEngine
FeatureAssembler: [0:176] 规则计算 + [176:192] 规则降级
MarketSnapshot.MLSource = "fallback"
MarketSnapshot.MLReady = false
```

**系统完全可用**，所有 192 维都有值。

### Phase 1: 接入第一个 ML 模型

```
MLEngine = ONNXMLEngine("market_regime_v1.onnx")
FeatureAssembler: [0:176] 规则计算 + [176:180] ML推理 + [180:192] 规则降级
MarketSnapshot.MLSource = "onnx_v1"
MarketSnapshot.MLReady = true
```

- ML 只覆盖 MarketRegime 4 维，其余继续降级
- 量化大脑代码**零修改**

### Phase 2: ML 全覆盖

```
MLEngine = FullMLEngine (市场分类 + 波动率预测 + 异常检测)
FeatureAssembler: [0:176] 规则计算 + [176:192] 全 ML 推理
MarketSnapshot.MLSource = "full_ml_v2"
```

- 量化大脑代码**依然零修改**
- 可开启 fallback.always_compute 对比 ML 输出和规则降级的差异

### Phase 3: ML 反哺规则特征（可选）

```
MLEngine 输出可能覆盖部分 [0:176] 维度（如用 LSTM 替代增量 EMA）
需要 FeatureAssembler 支持"维度级覆盖"配置
```

这一步是可选的长期演进，不在初期路线图内。
