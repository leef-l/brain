## 四、开发分期

### Sprint 1 — 采集管道（目标：WebSocket 连上 OKX 拿到数据）

**预计工作量：3-4 天**

```
步骤 1.1  Provider 接口定义
          provider/provider.go
          - DataProvider 接口
          - DataEvent / DataSink / EventPriority
          - ProviderHealth
          纯接口，无外部依赖

步骤 1.2  OKX 消息解析
          provider/okx_parse.go + okx_parse_test.go
          - parseCandlePayload(data []byte) → Candle
          - parseTradePayload(data []byte) → Trade
          - parseOrderBookPayload(data []byte) → OrderBook
          - parseFundingPayload(data []byte) → FundingRate
          纯函数，用 OKX 真实 JSON 样本写测试

步骤 1.3  OKX WebSocket 连接管理
          provider/okx_ws.go + okx_ws_test.go
          - 连接 wss://ws.okx.com:8443/ws/v5/public
          - 自动获取全量品种列表（GET /api/v5/public/instruments?instType=SWAP）
          - 订阅 candle/trades/books5/funding/oi
          - 断线重连（指数退避: 1s → 2s → 4s → 8s → 16s → 30s）
          - ping/pong 心跳保活
          依赖: gorilla/websocket

步骤 1.4  事件路由
          router/router.go
          - Realtime channel（无缓冲，tick/深度）
          - NearRT channel（缓冲 1024，K 线/资金费率）

步骤 1.5  集成验证
          写一个 cmd/data-debug/main.go 临时入口:
          - 连 OKX WebSocket
          - 打印收到的消息到 stdout
          - 验证解析正确性
          跑 5 分钟看数据流是否稳定
```

**Sprint 1 交付物**：能连上 OKX、收到实时行情、正确解析为 DataEvent。

### Sprint 3 — 数据处理器 + 增量指标（目标：K 线窗口 + 技术指标就绪）

**预计工作量：3-4 天**

```
步骤 3.1  增量指标实现
          processor/incremental.go + incremental_test.go
          每个指标独立结构体，实现 Update(price) + Value() float64

          IncrementalEMA:
            - period 参数（9/21/55）
            - multiplier = 2 / (period + 1)
            - Update: ema = price * mult + prevEma * (1 - mult)

          IncrementalRSI:
            - period 14
            - 分别追踪 avgGain / avgLoss
            - Update: 先判断涨跌，滚动更新

          IncrementalMACD:
            - ema12 + ema26 + signal(ema9 of diff)
            - Histogram() = diff - signal

          IncrementalATR:
            - period 14
            - True Range = max(H-L, |H-prevC|, |L-prevC|)
            - 滚动平均

          IncrementalADX:
            - +DI / -DI / ADX
            - 最复杂，需要追踪多个滚动值

          IncrementalBB:
            - 20 周期 SMA + 2 标准差
            - Position = (price - lower) / (upper - lower)

          测试: 用已知数据序列验证每个指标的数值正确性

步骤 3.2  K 线聚合器
          processor/candle_agg.go + candle_agg_test.go
          - CandleWindow: 每品种每时间框架一个窗口
          - History: 500 根滚动窗口（RingSlice）
          - OnCandle(): 新 K 线到达 → rotate → 更新所有增量指标
          - 启动时从 PG 加载历史 500 根填充 History

步骤 3.3  订单簿追踪器
          processor/orderbook.go + orderbook_test.go
          - 5 档 Bids/Asks
          - 派生: Imbalance / Spread / MidPrice / BidDepth / AskDepth
          - OnOrderBook(): 每次推送更新

步骤 3.4  逐笔成交分析器
          processor/tradeflow.go + tradeflow_test.go
          - 5 分钟滑动窗口
          - buyVolume / sellVolume / bigBuy / bigSell / tradeCount
          - Toxicity() / BigBuyRatio() / TradeDensityRatio()
          - OnTrade(): 每笔成交更新，定期清理过期数据
```

**Sprint 3 交付物**：所有技术指标可用，K 线窗口填充完毕，订单簿/成交流实时更新。

### Sprint 4 — 特征向量 + Ring Buffer（目标：Quant Brain 可消费）

**预计工作量：3-4 天**

```
步骤 4.1  MarketSnapshot 结构定义
          ringbuf/snapshot.go
          - 固定 1792 字节
          - Header + 价格 + 微观结构 + 特征向量 + padding
          - 确保内存对齐

步骤 4.2  Ring Buffer 实现
          ringbuf/ringbuf.go + ringbuf_test.go
          - 基于 /dev/shm mmap
          - RingHeader: WriteSeq(atomic) + SlotCount(1024) + SlotSize(1792)
          - Write(): atomic 递增 WriteSeq，写入对应 slot
          - 一写多读，无锁（SeqLock 语义）

          ringbuf/reader.go
          - Quant Brain 用的只读端
          - Read(): 读最新 snapshot，检查 SeqNum 一致性
          - 如果读到一半被覆盖（torn read），重试

步骤 4.3  特征向量引擎
          feature/engine.go + engine_test.go
          - 输入: CandleAggregator + OrderBookTracker + TradeFlowAnalyzer
          - 输出: [192]float64
          - 每秒计算一次，写入 Ring Buffer
          - 每分钟持久化一次到 PG vectors 表

          feature/vector.go
          - Serialize / Deserialize（[]float64 ↔ []byte）
          - Normalize（归一化工具）

步骤 4.4  DataBrain 完整版
          brain.go 完善
          启动流程:
            1. 加载配置
            2. 初始化 PG 连接池 + 建表
            3. 刷新活跃品种列表
            4. 历史回填（首次 5-10 分钟，后续秒级）
            5. 从 PG 加载 K 线历史 → 填充 CandleWindow
            6. 初始化 Ring Buffer（309 品种）
            7. 初始化 Feature Engine
            8. 连接 OKX WebSocket
            9. 主循环: WS → Validator → Router → Processor → Feature → RingBuf + PG

步骤 4.5  端到端测试
          brain_test.go
          - 用 ReplayProvider 灌历史数据
          - 验证 Ring Buffer 里的 MarketSnapshot 字段正确
          - 验证特征向量 192 维非零（除预留位）
```

**Sprint 4 交付物**：Data Brain 完整可运行，Quant Brain 可通过 Ring Buffer 读取实时数据。

## 五、接口契约（与 Quant Brain 的边界）

Quant Brain 只需要两个东西：

### 5.1 Ring Buffer（实时消费）

```go
// Quant Brain 这样读:
reader := ringbuf.NewReader("/dev/shm/brain-data-BTC-USDT-SWAP")
snap, err := reader.Latest()  // 返回 MarketSnapshot

// 用到的字段:
snap.CurrentPrice
snap.BidPrice / AskPrice
snap.FundingRate
snap.OrderBookImbalance
snap.TradeFlowToxicity
snap.FeatureVector  // [192]float64 — 策略直接消费
```

### 5.2 PG candles 表（回测消费）

```sql
-- Quant Brain 回测时直接查:
SELECT * FROM candles
WHERE inst_id = 'BTC-USDT-SWAP' AND bar = '1m'
  AND ts BETWEEN $1 AND $2
ORDER BY ts;
```

**不需要 JSON-RPC、不需要 HTTP API** — Ring Buffer 零拷贝 + PG 直查，最简单最快。

## 六、风险与注意事项

### 6.1 OKX WebSocket 坑

| 问题 | 应对 |
|------|------|
| 30 秒无数据自动断连 | 实现 ping/pong 心跳（25s 间隔） |
| 订阅多品种时消息量大 | 用 goroutine 池解析，不阻塞读循环 |
| 推送的 K 线可能有修正 | 同一 ts 的 K 线覆盖写入（UPSERT） |
| 断线期间丢数据 | 重连后触发 gap 检测 → REST 回填 |

### 6.2 OKX REST API 坑

| 问题 | 应对 |
|------|------|
| `/market/history-candles` 每次最多 100 根 | 分页，用 after 参数 |
| 限频 40 次/2s | rate.Limiter 控速 |
| 返回数据按时间倒序 | 接收后反转再写入 |
| 小币种可能缺少远期历史 | 跳过，不阻塞其他品种 |

### 6.3 共享内存坑

| 问题 | 应对 |
|------|------|
| 进程崩溃 shm 文件残留 | 启动时检测 Magic 字段，过期则重建 |
| 写入与读取竞争（torn read） | SeqLock：读前后比对 SeqNum，不一致则重试 |
| /dev/shm 空间不足 | 启动时检查，567MB 对 48G 不是问题 |

### 7.5 并发开发的注意事项

```
1. 接口先行
   provider/provider.go 和 store/store.go 里的接口定义
   必须在 Wave 1 第一天就确定，其他 Agent 依赖这些类型

2. 共享类型放 provider/provider.go
   DataEvent / Candle / Trade / OrderBook / FundingRate
   所有 Agent 都 import 这个包

3. 合并策略
   每个 Wave 结束后做一次合并:
   - go build ./brains/data/... 编译通过
   - go vet ./brains/data/... 无警告
   - go test ./brains/data/... 全过
   - 有冲突当场解决

4. 命名约定
   - 文件名: 小写下划线（candle_agg.go）
   - 包名: 目录名（provider / store / processor / feature / ringbuf）
   - 测试: 同目录 _test.go
   - 错误: fmt.Errorf("databrain: xxx: %w", err)
```
