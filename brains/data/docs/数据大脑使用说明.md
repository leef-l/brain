# 数据大脑 (Data Brain) 使用说明

## 概述

数据大脑是 Brain 系统的市场数据中枢，负责实时采集、校验、聚合加密货币合约行情，计算 192 维特征向量，为量化大脑提供决策输入。

**核心能力：**
- OKX 永续合约 WebSocket 实时行情采集（K线、订单簿、成交、资金费率）
- 自动发现高成交量合约（按 24h USDT 成交量排序）
- 历史数据回填（REST API，支持断点续传）
- 数据质量校验（价格跳变、时间戳异常、缺口检测）
- 增量技术指标计算（EMA/RSI/MACD/ATR/ADX/布林带）
- 192 维特征向量实时计算（1 秒刷新）
- Ring Buffer 内存快照（供量化大脑零延迟读取）
- PostgreSQL 持久化（可选，支持无数据库调试模式）

## 快速开始

### 最小启动（调试模式）

无需数据库，自动发现合约，数据仅存内存：

```bash
brain-data
```

### 指定合约

```bash
brain-data -instruments BTC-USDT-SWAP,ETH-USDT-SWAP,SOL-USDT-SWAP
```

### 带数据库 + 历史回填

```bash
brain-data \
  -pg "postgres://user:pass@localhost:5432/brain" \
  -backfill \
  -backfill-days 90
```

### 通过环境变量配置

```bash
export PG_URL="postgres://brain:secret@db:5432/brain"
brain-data -backfill
```

## 命令行参数

| 参数 | 环境变量 | 默认值 | 说明 |
|------|---------|--------|------|
| `-pg` | `PG_URL` | (空) | PostgreSQL 连接串，空则无持久化 |
| `-instruments` | — | (空) | 逗号分隔的合约列表，空则自动发现 |
| `-backfill` | — | `false` | 启动时是否回填历史数据 |
| `-backfill-days` | — | `90` | 回填天数 |

## 配置详解

完整配置示例见 `config.example.yaml`，以下是各模块说明。

### 活跃合约筛选

```yaml
active_list:
  min_volume_24h: 10000000       # 24h 最低成交量（USDT）
  max_instruments: 100           # 最多跟踪合约数
  update_interval: 168h          # 刷新周期（7天）
  always_include:                # 始终跟踪
    - BTC-USDT-SWAP
    - ETH-USDT-SWAP
    - SOL-USDT-SWAP
```

筛选逻辑：从 OKX REST API 拉取所有 SWAP 合约，按 24h USDT 成交量降序排列，取前 `max_instruments` 个，合并 `always_include`。

### 数据源

```yaml
providers:
  - name: okx-swap
    type: okx-swap
    params:
      ws_url: "wss://ws.okx.com:8443/ws/v5/public"
      rest_url: "https://www.okx.com"
      ping_interval: 25s
      reconnect_delay: [1s, 2s, 5s, 10s, 30s]
```

**订阅频道：**
- `candle1m` / `candle5m` / `candle15m` / `candle1H` / `candle4H` — K 线
- `books5` — 5 档订单簿
- `trades` — 逐笔成交
- `funding-rate` — 资金费率

**订阅限制：** 每条消息最多 100 个合约（OKX API 要求），自动分批发送。

**断线重连：** 指数退避（1s → 2s → 5s → 10s → 30s），自动重新订阅。

### 历史回填

```yaml
backfill:
  enabled: true
  max_days: 90
  batch_size: 100         # 每次请求的 K 线数
  rate_limit: 200ms       # 请求间隔
  concurrent: 3           # 并行 worker
```

**回填流程：**
1. 检查数据库中各合约各周期的最新时间戳
2. 从最新时间戳开始，向前回填到 `max_days` 天
3. 每批次 100 条，保存进度到 `backfill_progress` 表
4. 重启后自动从断点续传

**回填的周期：** `1m`、`5m`、`15m`、`1H`、`4H`

### 数据校验

```yaml
validation:
  max_price_jump: 0.10     # 单根 K 线最大涨跌幅 10%
  max_gap_duration: 5m     # 最大允许缺口
  stale_timeout: 60s       # 数据源静默超时
```

**校验规则：**

| 类型 | 级别 | 触发条件 | 处理 |
|------|------|---------|------|
| `price_spike` | critical | 单根 K 线涨跌超阈值 | 丢弃并告警 |
| `future_ts` | warning | 时间戳超前 >5s | 丢弃并告警 |
| `stale` | warning | 时间戳滞后 >5min | 丢弃并告警 |
| `gap` | warning | 连续 ≥3 根 K 线缺失 | 告警（不丢弃） |
| `duplicate_ts` | — | 重复时间戳 | 静默丢弃 |

### 内存环形缓冲区

```yaml
ring_buffer:
  candle_depth: 1000       # 每个（合约, 周期）保留 K 线数
  trade_depth: 5000        # 每个合约保留成交数
  order_book_depth: 100    # 每个合约保留订单簿快照数
```

### 特征计算

```yaml
feature:
  enabled: true
  windows: [5, 10, 20, 60]  # 滚动窗口（K 线根数）
  interval: 1s               # 计算周期
```

## 192 维特征向量

每秒为每个活跃合约计算一次，分为 6 个区块：

| 维度范围 | 数量 | 内容 |
|---------|------|------|
| `[0:60]` | 60 | 价格特征（5 个周期 × 12 维） |
| `[60:100]` | 40 | 成交量特征（预留） |
| `[100:130]` | 30 | 微观结构（订单簿失衡、价差、成交毒性） |
| `[130:160]` | 30 | 多尺度动量（EMA 偏离、RSI、MACD、布林带、ATR） |
| `[160:180]` | 20 | 跨品种（vs BTC、vs ETH） |
| `[180:192]` | 12 | 预留（宏观/情绪/链上） |

**每个周期的 12 维价格特征：**

| # | 特征 | 说明 |
|---|------|------|
| 1 | EMA9 偏离 | (价格 - EMA9) / EMA9 |
| 2 | EMA21 偏离 | (价格 - EMA21) / EMA21 |
| 3 | EMA55 偏离 | (价格 - EMA55) / EMA55 |
| 4 | EMA 交叉 | (EMA9 - EMA21) / EMA21 |
| 5 | RSI14 | 归一化到 [0, 1] |
| 6 | MACD 柱 | 归一化 |
| 7 | 布林带位置 | 在上下轨之间的位置 [0, 1] |
| 8 | ATR14 比率 | ATR / 价格 |
| 9 | 1 根涨跌幅 | 最近 1 根 K 线的涨跌幅 |
| 10 | 20 根涨跌幅 | 最近 20 根 K 线的涨跌幅 |
| 11 | 20 根波动率 | 标准差 / 均值 |
| 12 | ADX14 | 归一化到 [0, 1] |

## 数据库表结构

数据大脑启动时自动执行 `Migrate()`，创建以下表（幂等）：

### candles — K 线数据

```sql
CREATE TABLE candles (
    inst_id    VARCHAR(32),
    bar        VARCHAR(8),       -- 1m, 5m, 15m, 1H, 4H
    ts         BIGINT,           -- 开盘时间戳（毫秒）
    o, h, l, c DOUBLE PRECISION, -- OHLC
    vol        DOUBLE PRECISION, -- 成交量（张）
    vol_ccy    DOUBLE PRECISION, -- 成交量（USDT）
    PRIMARY KEY (inst_id, bar, ts)
);
```

### vectors — 特征向量

```sql
CREATE TABLE vectors (
    id         BIGSERIAL PRIMARY KEY,
    collection VARCHAR(32),      -- 默认 "default"
    inst_id    VARCHAR(32),
    timeframe  VARCHAR(8),
    ts         BIGINT,
    vector     BYTEA,            -- 192 × float64，小端序
    metadata   JSONB,
    created_at TIMESTAMPTZ
);
```

### backfill_progress — 回填进度

```sql
CREATE TABLE backfill_progress (
    inst_id   VARCHAR(32),
    timeframe VARCHAR(8),
    latest_ts BIGINT,
    bar_count INT,
    updated_at TIMESTAMPTZ,
    PRIMARY KEY (inst_id, timeframe)
);
```

## 运维指南

### 健康检查

数据大脑提供 `Health()` 方法返回运行状态：

```json
{
  "running": true,
  "ws_messages": 152389,
  "validator_rejected": 12,
  "ringbuf_writes": 48201,
  "pg_writes": 3600,
  "pg_errors": 0,
  "feature_compute_ms": 3,
  "active_instruments": 87
}
```

**关键指标：**
- `ws_messages` 持续增长 → WebSocket 正常
- `validator_rejected` 偶发正常，持续飙升 → 数据源异常
- `pg_errors` > 0 → 检查数据库连接
- `feature_compute_ms` > 100 → 合约数过多或机器性能不足

### 优雅关闭

发送 `SIGINT`（Ctrl+C）或 `SIGTERM`：

1. 取消所有协程上下文
2. 等待最多 10 秒完成进行中的写入
3. 关闭 WebSocket 连接
4. 关闭数据库连接池
5. 输出 `data brain stopped gracefully`

### 常见问题

**Q: 启动后没有数据流入？**
- 检查网络是否能连接 `wss://ws.okx.com:8443`
- 确认 `-instruments` 参数的合约名称正确（格式：`BTC-USDT-SWAP`）
- 查看日志中的 reconnect 信息

**Q: 回填速度很慢？**
- 调高 `backfill.concurrent`（默认 3）
- 缩短 `backfill.rate_limit`（注意不要触发 OKX 限频）
- 减少 `-backfill-days`

**Q: 内存占用过高？**
- 减少 `ring_buffer.candle_depth` / `trade_depth`
- 减少 `active_list.max_instruments`
- 检查 `feature.windows` 是否过多

**Q: 数据库写入失败？**
- 检查 PostgreSQL 连接串是否正确
- 确认 `Migrate()` 已成功执行（查看启动日志）
- 检查磁盘空间和连接数限制

## 与量化大脑对接

量化大脑通过 `Buffers()` 方法获取 Ring Buffer 管理器，零拷贝读取最新市场快照：

```go
// 获取 BTC 最新快照
snap, ok := dataBrain.Buffers().Latest("BTC-USDT-SWAP")
if ok {
    price := snap.CurrentPrice
    imbalance := snap.OrderBookImbalance
    features := snap.FeatureVector  // [192]float64
}

// 列出所有活跃合约
instruments := dataBrain.Buffers().Instruments()
```

**MarketSnapshot 字段：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `SeqNum` | uint64 | 单调递增序列号 |
| `InstID` | string | 合约 ID |
| `Timestamp` | int64 | 时间戳（毫秒） |
| `CurrentPrice` | float64 | 最新价 |
| `BidPrice` / `AskPrice` | float64 | 最优买卖价 |
| `FundingRate` | float64 | 当前资金费率 |
| `Volume24h` | float64 | 24h 成交量 |
| `OrderBookImbalance` | float64 | 订单簿失衡 [-1, 1] |
| `Spread` | float64 | 买卖价差比率 |
| `TradeFlowToxicity` | float64 | 成交毒性 [0, 1] |
| `BigBuyRatio` | float64 | 大单买入占比 |
| `TradeDensityRatio` | float64 | 成交密度 |
| `FeatureVector` | [192]float64 | 完整特征向量 |

## 部署示例

### Docker Compose

```yaml
version: "3.8"
services:
  postgres:
    image: postgres:16
    environment:
      POSTGRES_DB: brain
      POSTGRES_USER: brain
      POSTGRES_PASSWORD: secret
    volumes:
      - pgdata:/var/lib/postgresql/data

  data-brain:
    build: .
    command: >
      brain-data
        -pg "postgres://brain:secret@postgres:5432/brain?sslmode=disable"
        -backfill
        -backfill-days 90
    depends_on:
      - postgres
    restart: unless-stopped

volumes:
  pgdata:
```

### Systemd

```ini
[Unit]
Description=Brain Data Brain
After=postgresql.service

[Service]
Type=simple
Environment=PG_URL=postgres://brain:secret@localhost:5432/brain
ExecStart=/usr/local/bin/brain-data -backfill -backfill-days 30
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
```
