## 一、定位与边界

```yaml
brain:
  kind: data
  version: "1.0.0"
  scope: "OKX USDT 永续合约（~309 品种）"
  capabilities:
    - market_data.realtime       # WebSocket 实时行情
    - market_data.backfill       # REST API 历史回填
    - feature.compute            # 192 维特征向量
    - orderbook.snapshot         # 5 档深度
    - data.validation            # 数据质量守门
    - data.replay                # 回测回放
```

**当前只做 OKX 合约**。架构上通过 Provider 接口留口子，未来加 A 股/港股/美股/期货/农业时只需新增 Provider 实现，不动核心。

**数据大脑不做的事**：
- 不做交易决策（Quant Brain 的事）
- 不做下单执行（Execution Engine 的事）
- 不做风控判断（Risk Guard 的事）

## 三、Provider 接口（扩展口子）

```go
// DataProvider — 统一数据源接口
// 当前只有 OKX 实现，未来加新市场只需新增实现
type DataProvider interface {
    Name() string
    Start(ctx context.Context) error
    Stop(ctx context.Context) error
    Health() ProviderHealth
    Subscribe(sink DataSink) error
}

// DataSink — 数据消费端
type DataSink interface {
    OnEvent(event DataEvent)
}

// DataEvent — 统一数据事件
type DataEvent struct {
    Provider  string
    Symbol    string        // OKX 原始 ID: "BTC-USDT-SWAP"
    Topic     string        // "candle.1m.BTC-USDT-SWAP"
    Timestamp int64         // 数据产生时间（毫秒）
    LocalTS   int64         // 本地接收时间（纳秒）
    Priority  EventPriority
    Payload   any           // Candle / OrderBook / Trade
}

type EventPriority uint8
const (
    PriorityRealtime EventPriority = 1  // tick/深度 → Ring Buffer
    PriorityNearRT   EventPriority = 2  // K线/资金费率 → channel
)

type ProviderHealth struct {
    Status     string        // "healthy", "degraded", "disconnected"
    Latency    time.Duration
    LastEvent  time.Time
    ErrorCount int64
}
```

**为什么保留 Provider 接口而不直接写死 OKX？**
- 回测时用 ReplayProvider 替换 OKX Provider，Quant Brain 不需要改任何代码
- 未来加 Binance/A 股只需实现这个接口

## 四、OKX 合约 Provider

### 4.1 WebSocket 实时采集

```go
type OKXSwapProvider struct {
    config     OKXSwapConfig
    conn       *WSConnector
    backfiller *HistoryBackfiller
    eventSink  DataSink
}

type OKXSwapConfig struct {
    WSURL          string          // wss://ws.okx.com:8443/ws/v5/public
    Instruments    []string        // 空 = 自动拉全量 SWAP 品种
    ReconnectDelay []time.Duration // [1s, 2s, 4s, 8s, 16s, 30s]
}

// Start 流程:
// 1. GET /api/v5/public/instruments?instType=SWAP → 拿到全量品种列表
// 2. 历史回填（见 4.2）
// 3. 建立 WebSocket 连接
// 4. 订阅所有品种的以下频道
func (p *OKXSwapProvider) subscriptions() []WSSubscription {
    return []WSSubscription{
        {Channel: "candle1m",      InstIDs: p.instruments},
        {Channel: "candle5m",      InstIDs: p.instruments},
        {Channel: "candle15m",     InstIDs: p.instruments},
        {Channel: "candle1H",      InstIDs: p.instruments},
        {Channel: "candle4H",      InstIDs: p.instruments},
        {Channel: "books5",        InstIDs: p.instruments},  // 5 档深度, 100ms
        {Channel: "trades",        InstIDs: p.instruments},  // 逐笔, 实时
        {Channel: "funding-rate",  InstIDs: p.instruments},
        {Channel: "open-interest", InstIDs: p.instruments},
    }
}

// onWSMessage — 原始消息转统一 DataEvent
func (p *OKXSwapProvider) onWSMessage(channel, instID string, data []byte) {
    event := DataEvent{
        Provider: "okx_swap",
        Symbol:   instID,
        LocalTS:  time.Now().UnixNano(),
    }
    switch {
    case strings.HasPrefix(channel, "candle"):
        event.Topic    = fmt.Sprintf("candle.%s.%s", channel[6:], instID)
        event.Priority = PriorityNearRT
        event.Payload  = parseCandlePayload(data)
    case channel == "books5":
        event.Topic    = fmt.Sprintf("orderbook.%s", instID)
        event.Priority = PriorityRealtime
        event.Payload  = parseOrderBookPayload(data)
    case channel == "trades":
        event.Topic    = fmt.Sprintf("trade.%s", instID)
        event.Priority = PriorityRealtime
        event.Payload  = parseTradePayload(data)
    case channel == "funding-rate":
        event.Topic    = fmt.Sprintf("funding.%s", instID)
        event.Priority = PriorityNearRT
        event.Payload  = parseFundingPayload(data)
    }
    p.eventSink.OnEvent(event)
}
```

### 4.2 历史回填（HistoryBackfiller）

```go
type HistoryBackfiller struct {
    client    *http.Client
    store     CandleStore
    rateLimit *rate.Limiter  // OKX REST: 40 次/2s
}

type BackfillConfig struct {
    Timeframes []string      // ["1m", "5m", "15m", "1H", "4H"]
    MaxBars    int           // 每次请求最多 300 根
    GoBack     time.Duration // 回填多久: 90 天
}
```

**回填流程（首次启动）**：

```
1. 遍历活跃品种列表（~100 个）
2. 对每个品种、每个时间框架:
   a. 查 PG 里最新一条 K 线的时间戳
   b. 从该时间戳到现在，分页拉取（每页 300 根）
   c. OKX API: GET /api/v5/market/candles?instId=BTC-USDT-SWAP&bar=1m&limit=300&after={ts}
   d. 写入 PG candles 表
3. 限速: 40 次/2s（OKX REST 限频）
```

**回填数据量估算（100 个活跃品种 × 90 天）**：

| 时间框架 | 每品种 K 线数 | 总行数 | PG 大小 |
|---------|-------------|--------|---------|
| 1m | 129,600 | 12,960,000 | ~1.3 GB |
| 5m | 25,920 | 2,592,000 | ~260 MB |
| 15m | 8,640 | 864,000 | ~86 MB |
| 1H | 2,160 | 216,000 | ~22 MB |
| 4H | 540 | 54,000 | ~5 MB |
| **合计** | | **16,686,000** | **~1.7 GB** |

**回填耗时估算**：
- 100 品种 × 5 时间框架 = 500 个回填任务
- 每个任务平均 3-5 次 API 调用（分页）
- 500 × 4 = 2000 次调用 ÷ 20 次/s = **~100 秒 ≈ 2 分钟**
- 加上网络延迟和写入，实际 **5-10 分钟**完成

```go
func (b *HistoryBackfiller) BackfillAll(ctx context.Context, instruments []string) error {
    for _, instID := range instruments {
        for _, tf := range b.config.Timeframes {
            if err := b.backfillOne(ctx, instID, tf); err != nil {
                log.Warn("backfill failed", "inst", instID, "tf", tf, "err", err)
                continue  // 单品种失败不影响其他
            }
        }
    }
    return nil
}

func (b *HistoryBackfiller) backfillOne(ctx context.Context, instID, tf string) error {
    // 查 PG 里最新时间戳
    latestTS, err := b.store.LatestTimestamp(ctx, instID, tf)
    if err != nil {
        return err
    }

    // 如果没有数据，从 90 天前开始
    if latestTS == 0 {
        latestTS = time.Now().Add(-b.config.GoBack).UnixMilli()
    }

    // 分页拉取
    for {
        b.rateLimit.Wait(ctx)

        candles, err := b.fetchCandles(ctx, instID, tf, latestTS, b.config.MaxBars)
        if err != nil {
            return err
        }
        if len(candles) == 0 {
            break
        }

        if err := b.store.BatchInsert(ctx, candles); err != nil {
            return err
        }

        latestTS = candles[len(candles)-1].Timestamp
    }
    return nil
}
```

### 5.1 WebSocket 推送频率

| 频道 | 推送频率 | 说明 |
|------|---------|------|
| `trades` | **实时逐笔** | 有成交就推，ms 级，无聚合延迟 |
| `bbo-tbt` | **10ms** | OKX 最快频道（当前未订阅，备用） |
| `books5` | 100ms | 5 档快照，当前采用 |
| `books` | 100ms | 全深度增量 |
| `candle1m` | ~1s | K 线聚合推送 |
| `funding-rate` | 按事件 | 每 8 小时一次 |
| `open-interest` | ~3s | 持仓量变化 |

### 5.2 REST API 限频

| 接口 | 限频 | 用途 |
|------|------|------|
| `/market/candles` | 40 次/2s | 历史回填 |
| `/market/tickers` | 20 次/2s | 活跃品种刷新 |
| `/public/instruments` | 20 次/2s | 品种列表 |
| 下单 | 60 次/2s | Execution Engine 用，不是 Data Brain |

### 5.3 量化大脑消费精度：100ms（10Hz）够用

| 策略 | 需要的精度 | 原因 |
|------|-----------|------|
| 趋势跟踪 | 秒级 | 看 K 线形态 |
| 均值回归 | 秒级 | 看偏离度 |
| 突破动量 | 秒级 | 看收盘价 |
| 订单流 | 100ms | `books5` 本身 100ms |
| 止损/止盈 | **交易所侧** | OKX 条件单 `triggerPx` 由交易所执行 |

**不需要 ms 级监控**。ms 级只在高频做市场景有意义。

## 六、数据质量校验（DataValidator）

```go
type DataValidator struct {
    lastPrice   map[string]float64
    lastTS      map[string]int64
    gapDetector *GapDetector
    alertSink   func(alert ValidatorAlert)
}

type ValidatorAlert struct {
    Level   string // "warning", "critical"
    Type    string // "price_spike", "gap", "stale"
    Symbol  string
    Detail  string
}

func (v *DataValidator) Validate(event *DataEvent) bool {
    // 1. 时间戳 — 不接受未来数据或 5 分钟前的旧数据
    now := time.Now().UnixMilli()
    if event.Timestamp > now+5000 || event.Timestamp < now-300000 {
        return false
    }

    // 2. 价格突变 — 单 tick >10% 丢弃
    if price := extractPrice(event); price > 0 {
        if last, ok := v.lastPrice[event.Symbol]; ok && last > 0 {
            if math.Abs(price-last)/last > 0.10 {
                v.alert("critical", "price_spike", event.Symbol, "...")
                return false
            }
        }
        v.lastPrice[event.Symbol] = price
    }

    // 3. 重复时间戳 — 丢弃
    if last, ok := v.lastTS[event.Symbol]; ok && event.Timestamp <= last {
        return false
    }
    v.lastTS[event.Symbol] = event.Timestamp

    // 4. K 线 gap — 连续缺失 >3 根触发 REST 回填
    v.gapDetector.Observe(event)
    return true
}
```

校验配置：
```yaml
validation:
  max_price_change_pct: 10.0
  max_future_ts_ms: 5000
  max_stale_ts_ms: 300000
  gap_backfill_threshold: 3
```

## 七、Event Router

```go
type EventRouter struct {
    realtimeCh chan DataEvent  // 无缓冲，tick/深度
    nearRTCh   chan DataEvent  // 缓冲 1024，K 线/资金费率
}

func (r *EventRouter) OnEvent(event DataEvent) {
    switch event.Priority {
    case PriorityRealtime:
        select {
        case r.realtimeCh <- event:
        default:
            // tick 丢一个无影响，metric++
        }
    case PriorityNearRT:
        r.nearRTCh <- event
    }
}
```

### 8.1 K 线聚合器

```go
type CandleAggregator struct {

type CandleWindow struct {
    InstID    string
    Timeframe string
    Current   Candle
    History   *RingSlice[Candle]  // 固定 500 根滚动窗口

    // 增量指标（随 tick 更新，不用每次全量重算）

func (w *CandleWindow) OnTrade(price, qty float64, ts int64) {
    if ts >= w.Current.CloseTime {
        w.rotate()  // 切换到新 K 线，旧 K 线入 History
    }
    w.Current.Update(price, qty)
    w.ema9.Update(price)
    w.rsi14.Update(price)
    // ... 其他指标增量更新
}
```

### 8.3 逐笔成交分析器

```go
type TradeFlowAnalyzer struct {
    windows map[string]*FlowWindow  // instID → 滑动 5 分钟窗口
}

type FlowWindow struct {
    trades     *RingSlice[TradeEvent]
    buyVolume  float64
    sellVolume float64
    bigBuyVol  float64  // 大单（> 均值 3 倍）
    bigSellVol float64
    tradeCount int
}

func (w *FlowWindow) Toxicity() float64 {
    total := w.buyVolume + w.sellVolume
    if total == 0 { return 0 }
    return math.Abs(w.buyVolume-w.sellVolume) / total
```

## 九、特征向量引擎（192 维）

```go
type FeatureEngine struct {
```

### 9.1 向量组成

```
[0:60]    价格特征      5 个时间框架 × 12 维
[60:100]  成交量特征    量比、OBV、大单占比、成交密度
[100:130] 微观结构      订单簿不平衡、Spread、Toxicity、资金费率
[130:160] 时序特征      多尺度价格变化率、波动率、ATR 比率
[160:180] 跨品种特征    BTC/ETH 相关性（按 BTC 为基准）
[180:192] 预留扩展      当前填 0，未来加宏观/情绪/链上
```

### 9.2 价格特征计算（每时间框架 12 维）

```go
func (e *FeatureEngine) computeFeatureVector(instID string) []float64 {
    vec := make([]float64, e.vectorDim)

    for i, tf := range []string{"1m", "5m", "15m", "1H", "4H"} {
        w := e.candles.GetWindow(instID, tf)
        if w == nil { continue }
        base := i * 12
        price := w.Current.Close

        vec[base+0]  = (price - w.ema9.Value()) / price    // EMA9 偏离
        vec[base+1]  = (price - w.ema21.Value()) / price   // EMA21 偏离
        vec[base+2]  = (price - w.ema55.Value()) / price   // EMA55 偏离
        vec[base+3]  = (w.ema9.Value() - w.ema21.Value()) / price  // EMA 交叉
        vec[base+4]  = w.rsi14.Value() / 100.0             // RSI 归一化
        vec[base+5]  = w.macd.Histogram() / price          // MACD 柱
        vec[base+6]  = w.bbPosition()                      // 布林带位置
        vec[base+7]  = w.atr14.Value() / price             // ATR 比率
        vec[base+8]  = w.priceChangeRate(5)                // 5 根变化率
        vec[base+9]  = w.priceChangeRate(20)               // 20 根变化率
        vec[base+10] = w.volatility(20)                    // 20 根波动率
        vec[base+11] = w.adx.Value() / 100.0              // ADX 归一化
    }

    // [60:100] 成交量 ...
    // [100:130] 微观结构 ...
    // [130:160] 时序 ...
    // [160:180] 跨品种（BTC/ETH 联动）...
    // [180:192] 预留 0

    return vec
}
```

### 9.3 特征向量依赖链

```
特征向量计算需要的原始数据:

价格特征 [0:60]    ← K 线 OHLC（5 个时间框架 × 500 根历史）
成交量   [60:100]  ← 逐笔成交 + 分钟成交量
微观结构 [100:130] ← 订单簿 5 档 + 逐笔成交 + 资金费率
时序     [130:160] ← K 线历史序列
跨品种   [160:180] ← 多品种同时有 K 线数据
预留     [180:192] ← 当前填 0

所以: 必须先采集到足够的 K 线数据，才能计算特征向量。
历史回填 90 天 K 线后，所有维度（除预留）立刻可算。
```

```go
type SharedRingBuffer struct {

```

**数据流精度链路**:

```
OKX WebSocket（ms 级推送）
    │
    ▼
Data Brain 接收（保留 ms 时间戳）
    │
    ▼
Ring Buffer 写入（ms 精度，全量 309 品种）
    │
    ├──► Quant Brain 消费（100ms / 10Hz tick loop）
    │
    └──► PostgreSQL 持久化（分钟级，仅活跃品种）
```

**设计原则：接收保留最高精度，消费按需降级。**

### 11.2 活跃品种列表

```go
type ActiveInstrumentList struct {
    mu         sync.RWMutex
    active     map[string]bool
    lastUpdate time.Time
    config     ActiveListConfig
}

type ActiveListConfig struct {
    MinVolume24h   float64       // 最低 24h 成交额（USDT），默认 10_000_000
    MaxInstruments int           // 最大品种数，默认 100
    UpdateInterval time.Duration // 更新间隔，默认 7 天
    AlwaysInclude  []string      // 强制包含: BTC/ETH/SOL
}

// 每周调 OKX ticker API，按成交额排序取 Top N
func (l *ActiveInstrumentList) Refresh(ctx context.Context) error {
    // GET /api/v5/market/tickers?instType=SWAP
    tickers := fetchOKXTickers(ctx)
    sort.Slice(tickers, func(i, j int) bool {
        return tickers[i].VolUsdt24h > tickers[j].VolUsdt24h
    })

    newActive := make(map[string]bool)
    // 先加强制包含的
    for _, id := range l.config.AlwaysInclude {
        newActive[id] = true
    }
    // 再按成交额填充
    for _, t := range tickers {
        if len(newActive) >= l.config.MaxInstruments { break }
        if t.VolUsdt24h < l.config.MinVolume24h { break }
        newActive[t.InstID] = true
    }

    l.mu.Lock()
    l.active = newActive
    l.lastUpdate = time.Now()
    l.mu.Unlock()
    return nil
}

func (l *ActiveInstrumentList) IsActive(instID string) bool {
    l.mu.RLock()
    defer l.mu.RUnlock()
    return l.active[instID]
}
```

## 十二、回测 Provider

```go
type ReplayProvider struct {
    store     CandleStore
    speed     float64  // 1.0 = 实时速度, 0 = 尽速回放
    eventSink DataSink
}

func (p *ReplayProvider) Replay(ctx context.Context, from, to time.Time, speed float64) error {
    candles, _ := p.store.QueryRange(ctx, from, to)
    var prevTS int64
    for _, c := range candles {
        if speed > 0 && prevTS > 0 {
            delay := time.Duration(float64(c.Timestamp-prevTS)*1e6) / time.Duration(speed)
            time.Sleep(delay)
        }
        prevTS = c.Timestamp
        p.eventSink.OnEvent(DataEvent{
            Provider:  "replay",
            Symbol:    c.InstID,
            Topic:     fmt.Sprintf("candle.%s.%s", c.Bar, c.InstID),
            Timestamp: c.Timestamp,
            Priority:  PriorityNearRT,
            Payload:   c,
        })
    }
    return nil
}
```

**Quant Brain 不需要改任何代码** — 它只消费 Ring Buffer 里的 MarketSnapshot，不关心数据来自实时还是回放。
