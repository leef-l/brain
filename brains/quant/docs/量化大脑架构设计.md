## 一、定位与边界

### 量化大脑是什么

量化大脑是交易系统的**决策层**，类比人的"前额叶 + 运动皮层"。

它消费数据大脑输出的增强特征向量（192 维），做出"做多/做空/观望"的决策，并管理风险和执行。

### 职责划分

| 做 | 不做 |
|----|------|
| 消费特征向量，生成交易信号 | 数据采集 |
| 信号聚合与冲突消解 | 特征计算 |
| 三层风控 | **ML 推理**（这是数据大脑的职责） |
| 仓位计算（Kelly / Bayesian） | 模型训练 |
| 执行管理（Paper / Live） | 用户交互 |
| 回测验证 | |
| 通过 LLM 做决策审查（非推理） | |

### 与数据大脑的关系

```
数据大脑 (感知层)                    量化大脑 (决策层)
┌────────────────────┐              ┌────────────────────┐
│ 采集 → 处理 → 特征  │              │ 信号 → 聚合 → 风控  │
│                    │  Ring Buffer  │                    │
│ ML推理 / 规则降级   │─────────────→│ 读取192维向量       │
│                    │  每秒1次      │ 不关心ML还是规则     │
│ 输出: 增强特征向量   │              │                    │
│ [0:176] 规则特征    │              │ 输出: 交易意图       │
│ [176:192] ML/降级   │              │                    │
└────────────────────┘              └────────────────────┘
```

**关键原则：量化大脑不做 ML 推理，不加载模型文件，不调用推理服务。**

数据大脑的 192 维向量已经包含了 ML 推理结果（或规则降级值）。量化大脑直接读取这些维度，无需知道它们是 ML 算的还是规则算的。

### 与中央大脑的关系

```
中央大脑 (Central)
  │
  ├── Orchestrator.Delegate("quant", instruction)
  │     → subtask.delegate RPC
  │
  ├── LLMProxy.ProviderFactory("quant")
  │     → 量化大脑通过 llm.complete 反向 RPC 调用 AI
  │     → 用于：决策审查 / 策略分析 / 权重优化（低频）
  │
  └── 配置注入:
        BrainRegistration.Model = "claude-sonnet-4-6"
```

**量化大脑是 sidecar 进程**，通过 stdio JSON-RPC 与中央大脑通信。
AI 模型能力由中央大脑的 LLMProxy 代理提供，量化大脑不持有 API key。

## 三、核心变更：策略从重算指标改为读特征向量

### 3.1 问题

当前 4 个策略（TrendFollower、MeanReversion、BreakoutMomentum、OrderFlow）全部从 `view.Candles()` 取原始 K 线，然后**自行重算** EMA/RSI/MACD/ADX/BB 等指标。

而数据大脑已经用**增量计算器**（O(1) 更新）精确计算了这些指标并写入了 192 维特征向量。

这导致：
1. **双重计算浪费** — 同样的指标算了两遍
2. **精度不一致** — 策略用批量回溯算，数据大脑用增量算，结果可能不同
3. **192 维向量成为死代码** — 没人消费
4. **MarketRegime 等 ML 维度无法被利用** — 策略根本不读向量

### 3.2 解决方案：FeatureView

新增 `FeatureView` 接口，策略通过它直接读取特征向量中的预计算值：

```go
// FeatureView 提供对 192 维特征向量的结构化读取。
// 策略通过这个接口消费数据大脑的计算结果，而非自行重算。
type FeatureView interface {
    // ---- 价格特征 [0:60] ----
    // tf: "1m", "5m", "15m", "1H", "4H"
    EMADeviation(tf string, period int) float64  // period: 9/21/55
    EMACross(tf string) float64                   // EMA9-EMA21
    RSI(tf string) float64                        // [0, 1]
    MACDHistogram(tf string) float64
    BBPosition(tf string) float64                 // [0, 1]
    ATRRatio(tf string) float64
    PriceChange(tf string, bars int) float64      // bars: 5/20
    Volatility(tf string, bars int) float64       // bars: 5/20
    ADX(tf string) float64                        // [0, 1]

    // ---- 成交量特征 [60:100] ----
    VolumeRatio(tf string) float64                // vol/SMA(vol,20)
    OBVSlope(tf string) float64
    VolumePriceCorr(tf string) float64            // [-1, 1]
    VolumeBreakout(tf string) bool                // 放量突破

    // ---- 微观结构 [100:130] ----
    OrderBookImbalance() float64                  // [-1, 1]
    Spread() float64
    TradeFlowToxicity() float64                   // [0, 1]
    BigBuyRatio() float64
    BigSellRatio() float64
    TradeDensityRatio() float64
    BuySellRatio() float64
    FundingRate() float64

    // ---- 动量 [130:160] ----
    Momentum(tf string, bars int) float64         // bars: 1/3/10
    VolatilityRatio(tf string) float64            // vol5/vol20

    // ---- 跨资产 [160:176] ----
    BTCExcessReturn() float64
    BTCMomentum() float64
    ETHMomentum() float64
    BTCCorrelation() float64

    // ---- ML 增强 / 规则降级 [176:192] ----
    MarketRegime() MarketRegimeProb               // 4 概率
    VolPrediction() VolPrediction                 // 波动率预测
    AnomalyScore() AnomalyScore                   // 异常评分

    // ---- 元信息 ----
    MLReady() bool                                // ML 是否在线
    Symbol() string
    CurrentPrice() float64
    FeatureVector() []float64                     // 原始 192 维
}

type MarketRegimeProb struct {
    Trend    float64  // [176]
    Range    float64  // [177]
    Breakout float64  // [178]
    Panic    float64  // [179]
}

func (m MarketRegimeProb) Dominant() string {
    max := m.Trend
    name := "trend"
    if m.Range > max { max = m.Range; name = "range" }
    if m.Breakout > max { max = m.Breakout; name = "breakout" }
    if m.Panic > max { name = "panic" }
    return name
}

type VolPrediction struct {
    Vol1H         float64  // [180]
    Vol4H         float64  // [181]
    VolPercentile float64  // [182]
    VolDirection  float64  // [183]
}

type AnomalyScore struct {
    Price     float64  // [184]
    Volume    float64  // [185]
    OrderBook float64  // [186]
    Combined  float64  // [187]
}
```

### 3.3 FeatureView 的实现

```go
// LiveFeatureView 从 MarketSnapshot 实现 FeatureView。
// 所有方法都是 O(1) 的索引读取，没有任何计算。
type LiveFeatureView struct {
    snap    ringbuf.MarketSnapshot
    vec     [192]float64  // 缓存的特征向量
}

// 时间框架索引
var tfIndex = map[string]int{
    "1m": 0, "5m": 1, "15m": 2, "1H": 3, "4H": 4,
}

func (v *LiveFeatureView) EMADeviation(tf string, period int) float64 {
    idx := tfIndex[tf]
    base := idx * 12
    switch period {
    case 9:  return v.vec[base+0]
    case 21: return v.vec[base+1]
    case 55: return v.vec[base+2]
    }
    return 0
}

func (v *LiveFeatureView) RSI(tf string) float64 {
    return v.vec[tfIndex[tf]*12 + 4]
}

func (v *LiveFeatureView) MarketRegime() MarketRegimeProb {
    return MarketRegimeProb{
        Trend:    v.vec[176],
        Range:    v.vec[177],
        Breakout: v.vec[178],
        Panic:    v.vec[179],
    }
}

// ... 其余方法都是类似的索引读取
```

## 四、策略 v2 — 读特征向量

### 4.1 TrendFollower v2

```go
func (TrendFollower) Compute(view MarketView) Signal {
    if !view.HasFeatureView() {
        return t.computeLegacy(view)  // 回测降级，使用原始 Candles
    }
    f := view.Feature()
    tf := view.Timeframe()  // 主时间框架 "1H"

    // 直接读取预计算指标（O(1)，无需重算）
    ema9dev  := f.EMADeviation(tf, 9)
    ema21dev := f.EMADeviation(tf, 21)
    ema55dev := f.EMADeviation(tf, 55)
    macdHist := f.MACDHistogram(tf)
    adx      := f.ADX(tf)              // 已归一化到 [0,1]
    atrRatio := f.ATRRatio(tf)
    emaCross := f.EMACross(tf)

    // EMA 排列判断：
    // 多头排列 = price > EMA9 > EMA21 > EMA55
    // 特征向量中: EMADeviation(9) > 0 && EMADeviation(21) > 0 && EMADeviation(55) > 0
    // 且 EMADeviation(9) > EMADeviation(21) > EMADeviation(55)（偏离逐级递增）
    bullish := ema9dev > 0 && ema21dev > 0 && ema55dev > 0 &&
        ema9dev > ema21dev &&
        (adx > 0.20 || macdHist > 0)  // ADX 已归一化，0.20 = 原来的 20

    bearish := ema9dev < 0 && ema21dev < 0 && ema55dev < 0 &&
        ema9dev < ema21dev &&
        (adx > 0.20 || macdHist < 0)

    if !bullish && !bearish {
        return Signal{
            Strategy:   "TrendFollower",
            Direction:  DirectionHold,
            Confidence: 0.1,
            Reason:     "trend conditions not aligned",
            Timestamp:  time.Now().UTC(),
        }
    }

    confidence := 0.35
    direction := DirectionHold
    reason := ""

    if bullish {
        direction = DirectionLong
        confidence += 0.35
        reason = fmt.Sprintf("ema alignment bullish, adx=%.2f, macd=%.4f", adx, macdHist)
    } else {
        direction = DirectionShort
        confidence += 0.35
        reason = fmt.Sprintf("ema alignment bearish, adx=%.2f, macd=%.4f", adx, macdHist)
    }

    // 多时间框架确认 — 直接读 4H 向量维度
    htf := "4H"
    htfEma9 := f.EMADeviation(htf, 9)
    htfEma21 := f.EMADeviation(htf, 21)
    htfEma55 := f.EMADeviation(htf, 55)
    if direction == DirectionLong && htfEma9 > 0 && htfEma21 > 0 && htfEma55 > 0 {
        confidence *= 1.3
        reason += "; 4H confirms trend"
    }
    if direction == DirectionShort && htfEma9 < 0 && htfEma21 < 0 && htfEma55 < 0 {
        confidence *= 1.3
        reason += "; 4H confirms trend"
    }

    // 融资费率确认
    if fr := f.FundingRate(); direction == DirectionLong && fr > 0 || direction == DirectionShort && fr < 0 {
        confidence *= 1.15
        reason += "; funding aligned"
    }

    // 历史相似度（保留接口，待实现）
    if winRate := view.SimilarityWinRate(); winRate > 0 {
        if direction == DirectionLong && winRate >= 0.55 {
            confidence *= 1.2
            reason += "; similar history supportive"
        }
    }

    confidence = clamp(confidence, 0, 1)
    priceNow := f.CurrentPrice()
    stopDistance := atrRatio * priceNow * 2
    takeDistance := atrRatio * priceNow * 4

    signal := Signal{
        Strategy:   "TrendFollower",
        Direction:  direction,
        Confidence: confidence,
        Entry:      priceNow,
        Reason:     reason,
        Timestamp:  time.Now().UTC(),
    }
    if direction == DirectionLong {
        signal.StopLoss = priceNow - stopDistance
        signal.TakeProfit = priceNow + takeDistance
    } else {
        signal.StopLoss = priceNow + stopDistance
        signal.TakeProfit = priceNow - takeDistance
    }
    return signal
}
```

### 4.2 其他策略改造原则

| 策略 | v1 自行计算 | v2 读取向量维度 |
|------|-----------|---------------|
| TrendFollower | ema(), macdHistogram(), adx(), atr() | [0:60] Price 区间 |
| MeanReversion | bollinger(), rsi(), adx(), sma() | BBPosition, RSI, ADX, Volatility |
| BreakoutMomentum | highestHigh(), lowestLow(), obvSlope() | PriceChange, VolumeRatio, OBVSlope |
| OrderFlow | view.OrderBookImbalance() 等标量 | [100:130] Microstructure 区间 |

**OrderFlow 策略变化最小** — 它本来就读标量值，v2 中改为从 FeatureView 读取对应索引。

**每个策略保留 `computeLegacy()` 方法**，供回测引擎使用（回测时没有实时特征向量，需要从历史 K 线重算）。

## 五、RegimeAwareAggregator — 市场状态感知聚合

这是 v3 最重要的新组件。它替代原来的 Aggregator，利用数据大脑提供的 MarketRegime 维度动态调整策略权重。

```go
// RegimeAwareAggregator 根据市场状态动态调整策略权重。
// 它读取 FeatureVector [176:179] 的 MarketRegime 概率，
// 自动偏向适合当前市况的策略。
type RegimeAwareAggregator struct {
    // 基础权重（配置文件设定）
    BaseWeights map[string]float64

    // 每种市场状态下各策略的权重修正系数
    // 例如：趋势市 → TrendFollower 权重 ×1.5, MeanReversion ×0.5
    RegimeModifiers map[string]map[string]float64

    // 置信度校准器（回测数据驱动）
    Calibrator *ConfidenceCalibrator

    // 冲突消解模式
    ConflictMode ConflictMode

    // 历史 Oracle（可选）
    Oracle HistoricalOracle

    // 阈值
    LongThreshold   float64
    ShortThreshold  float64
    DominanceFactor float64
}

// ConflictMode 定义策略信号冲突时的处理方式
type ConflictMode string

const (
    ConflictSuppress ConflictMode = "suppress"  // 冲突时 Hold
    ConflictWeighted ConflictMode = "weighted"  // 按权重投票
    ConflictVeto     ConflictMode = "veto"      // 任一策略反对则 Hold
)

// DefaultRegimeModifiers 返回默认的市场状态权重修正。
func DefaultRegimeModifiers() map[string]map[string]float64 {
    return map[string]map[string]float64{
        "trend": {
            "TrendFollower":    1.5,
            "MeanReversion":    0.5,
            "BreakoutMomentum": 1.2,
            "OrderFlow":        0.8,
        },
        "range": {
            "TrendFollower":    0.5,
            "MeanReversion":    1.5,
            "BreakoutMomentum": 0.8,
            "OrderFlow":        1.2,
        },
        "breakout": {
            "TrendFollower":    1.0,
            "MeanReversion":    0.3,
            "BreakoutMomentum": 1.8,
            "OrderFlow":        1.2,
        },
        "panic": {
            "TrendFollower":    0.3,
            "MeanReversion":    0.3,
            "BreakoutMomentum": 0.3,
            "OrderFlow":        0.5,
        },
    }
}

// Aggregate 生成聚合信号。
func (a *RegimeAwareAggregator) Aggregate(
    view MarketView,
    signals []Signal,
    review ReviewContext,
) AggregatedSignal {
    result := AggregatedSignal{Symbol: view.Symbol()}
    if len(signals) == 0 {
        result.Direction = DirectionHold
        result.RejectionReason = "no strategy signals"
        return result
    }

    // 1. 读取市场状态（来自特征向量 [176:179]）
    regime := "range"  // 默认震荡
    if view.HasFeatureView() {
        regime = view.Feature().MarketRegime().Dominant()
    }
    result.MarketRegime = regime

    // 2. 计算当前市况下的有效权重
    effectiveWeights := a.effectiveWeights(regime)

    // 3. 加权聚合
    for _, signal := range signals {
        if signal.Direction != DirectionLong && signal.Direction != DirectionShort {
            continue
        }

        weight := effectiveWeights[signal.Strategy]
        calibrated := signal.Confidence
        if a.Calibrator != nil {
            calibrated = a.Calibrator.Calibrate(signal.Strategy, signal.Confidence)
        }

        score := clamp(calibrated, 0, 1) * weight
        if signal.Direction == DirectionLong {
            result.LongScore += score
        } else {
            result.ShortScore += score
        }
        result.Signals = append(result.Signals, signal)
    }

    // 4. 冲突消解
    hasLong := result.LongScore > 0
    hasShort := result.ShortScore > 0
    if hasLong && hasShort && a.ConflictMode == ConflictSuppress {
        // 双方都有信号但差距不大 → Hold
        ratio := result.LongScore / math.Max(result.ShortScore, 1e-10)
        if ratio < a.DominanceFactor && ratio > 1.0/a.DominanceFactor {
            result.Direction = DirectionHold
            result.RejectionReason = fmt.Sprintf(
                "conflict suppressed: long=%.3f short=%.3f regime=%s",
                result.LongScore, result.ShortScore, regime)
            return result
        }
    }

    // 5. 恐慌模式特殊处理
    if regime == "panic" {
        // 恐慌时大幅提升开仓门槛
        result.Direction = DirectionHold
        result.RejectionReason = "panic regime: all new positions suppressed"
        result.NeedsReview = true
        result.ReviewReason = "panic regime detected"
        return result
    }

    // 6. 方向决策
    switch {
    case result.LongScore > a.LongThreshold && result.LongScore > result.ShortScore*a.DominanceFactor:
        result.Direction = DirectionLong
    case result.ShortScore > a.ShortThreshold && result.ShortScore > result.LongScore*a.DominanceFactor:
        result.Direction = DirectionShort
    default:
        result.Direction = DirectionHold
        result.RejectionReason = "signals did not cross directional threshold"
    }

    // 7. Oracle 历史校验（可选）
    if result.Direction != DirectionHold && a.Oracle != nil {
        if winRate, ok := a.Oracle.HistoricalWinRate(
            view.Symbol(), result.Direction, view.FeatureVector()); ok {
            if result.Direction == DirectionLong && winRate < 0.35 {
                result.Direction = DirectionHold
                result.RejectionReason = "historical long win rate below 0.35"
            }
            if result.Direction == DirectionShort && winRate > 0.65 {
                result.Direction = DirectionHold
                result.RejectionReason = "historical short win rate above 0.65"
            }
        }
    }

    // 8. 置信度
    if result.Direction == DirectionLong {
        result.Confidence = result.LongScore
    } else if result.Direction == DirectionShort {
        result.Confidence = result.ShortScore
    } else {
        result.Confidence = 0
    }

    // 9. 审查触发
    if review.OpenPositions >= 3 {
        result.NeedsReview = true
        result.ReviewReason = "open positions >= 3"
    }
    if review.DailyLossPct > 3 {
        result.NeedsReview = true
        result.ReviewReason += "; daily loss > 3%"
    }
    if review.AnomalyDetected {
        result.NeedsReview = true
        result.ReviewReason += "; anomaly detected"
    }

    return result
}

// effectiveWeights 根据市场状态修正基础权重。
func (a *RegimeAwareAggregator) effectiveWeights(regime string) map[string]float64 {
    effective := make(map[string]float64, len(a.BaseWeights))
    modifiers := a.RegimeModifiers[regime]

    totalWeight := 0.0
    for name, base := range a.BaseWeights {
        mod := 1.0
        if modifiers != nil {
            if m, ok := modifiers[name]; ok {
                mod = m
            }
        }
        effective[name] = base * mod
        totalWeight += effective[name]
    }

    // 归一化，保证权重总和 = 1.0
    if totalWeight > 0 {
        for name := range effective {
            effective[name] /= totalWeight
        }
    }
    return effective
}
```

### 6.3 BayesianSizer — 贝叶斯 Kelly

```go
// BayesianSizer 在样本不足时用保守先验，随数据积累逐步逼近最优 Kelly。
type BayesianSizer struct {
    PositionSizer               // 嵌入现有 Sizer
    PriorAlpha    float64       // Beta 先验 α（默认 2）
    PriorBeta     float64       // Beta 先验 β（默认 2）
    MinObservations int          // 最少观察次数（默认 30）
}

// Size 计算仓位大小。
func (s *BayesianSizer) Size(req SizeRequest, tradeCount int, wins int) (SizeResult, error) {
    if tradeCount < s.MinObservations {
        // 样本不足 → 用固定小仓位替代 Kelly
        return SizeResult{
            RiskFraction: s.MinFraction,
            Notional:     req.AccountEquity * s.MinFraction,
        }, nil
    }

    // 后验均值: (α + wins) / (α + β + total)
    posteriorWinRate := (s.PriorAlpha + float64(wins)) /
        (s.PriorAlpha + s.PriorBeta + float64(tradeCount))

    req.WinRate = posteriorWinRate
    return s.PositionSizer.Size(req)
}
```

## 十四、设计原则

1. **不做推理，只做决策** — ML 推理是数据大脑的职责，量化大脑只读结果
2. **特征向量是唯一数据源** — 策略通过 FeatureView 读取预计算值，不重算
3. **无 ML 完全可用** — 规则降级保证所有 192 维都有值
4. **有 ML 零修改** — 向量维度的语义不变，来源透明
5. **先回测后上线** — Sharpe < 0.8 不上线
6. **恐慌保护** — MarketRegime = panic 时暂停所有开仓
7. **贝叶斯保守** — 样本不足时用固定小仓位，不盲用 Kelly
8. **配置驱动** — 权重/阈值/风控限额全部通过 YAML 配置
9. **可追溯** — 每笔交易记录市场状态、策略、置信度、特征哈希
