# 35. 项目级记忆与多项目管理(MACCS Wave 7+)

> **版本**: v1.0
> **创建日期**: 2026-05-02
> **实现位置**:
> - `sdk/persistence/projects_store.go` — 项目元数据(workdir + name + last_active)
> - `sdk/persistence/project_memory_store.go` — 项目记忆 entries(decision/lesson/pattern…)

> - `cmd/brain/cmd_serve.go` — POST /v1/runs body `project_id` / `project_name`

## 0. 设计目标

> **一个工作目录(workdir)可以对应多个项目(project)**,每个项目有独立 ID、对话历史、项目记忆。chat 启动时让用户选/建/跳过持久化。run 与 HTTP API 显式传 project name 或 id。

### 与原有"单一 ProjectMemory"的差异

| 维度 | 旧设计 | 新设计(Wave 7+) |
|------|-------|----------------|
| 默认实现 | `MemProjectMemory`(进程退出丢) | `SQLiteProjectMemoryStore`(持久) |
| 一个 workdir | 一个隐式项目 | N 个显式项目 |
| 项目切换 | 不支持 | `/project switch` |
| 对话历史 | 不写 ProjectStore | 自动写入 `project_conversations` |

## 1. 数据库 schema

### 1.1 `projects` 表(新增)

```sql
CREATE TABLE projects (
    id              TEXT PRIMARY KEY,        -- "abc123def456" 12 位 hex 随机
    workdir         TEXT NOT NULL,            -- 绝对路径 "/home/u/proj1"
    name            TEXT NOT NULL,            -- 用户可读名 "todo-app"
    created_at      TEXT NOT NULL,
    last_active_at  TEXT NOT NULL,
    metadata_json   TEXT
);
CREATE INDEX idx_projects_workdir ON projects(workdir);
CREATE UNIQUE INDEX idx_projects_workdir_name ON projects(workdir, name);
```

> **(workdir, name) 唯一约束**:同一目录里两个项目不能重名;不同目录可以同名。

### 1.2 `project_memory` 表(新增)

```sql
CREATE TABLE project_memory (
    id          TEXT PRIMARY KEY,
    project_id  TEXT NOT NULL,
    type        TEXT NOT NULL,    -- decision/lesson/pattern/preference/...
    content     TEXT NOT NULL,
    summary     TEXT,
    tags_json   TEXT,
    importance  REAL,
    created_at  TEXT NOT NULL,
    expires_at  TEXT
);
CREATE INDEX idx_pm_project_imp ON project_memory(project_id, importance DESC);
```

### 1.3 `project_conversations` 表(已有,未变)

```sql
CREATE TABLE project_conversations (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    project_id      TEXT NOT NULL,
    role            TEXT NOT NULL,
    content_json    TEXT NOT NULL,
    created_at      TEXT NOT NULL
);
CREATE INDEX idx_project_conv_project ON project_conversations(project_id);
```

## 2. 三种入口的项目解析

| 模式 | 项目 ID 来源 | 找不到时 |
|------|-------------|---------|
| `brain chat` 默认启动 | **强制弹选择器**:已有项目列表 / 新建 / 跳过 | 用户输入新建 |
| `brain chat --project NAME` | (workdir, NAME) 找/建 | 自动新建 |
| `brain chat --new-project NAME` | 强制新建 | — |
| `brain chat --no-project` | 跳过 | 无项目模式 |
| `brain run --project NAME` | (workdir, NAME) 找/建 | 自动新建 |
| `brain run`(无 flag) | **不持久化**(向后兼容) | — |
| `POST /v1/runs body.project_id` | ID 直查 | 报告找不到,跳过持久化 |
| `POST /v1/runs body.project_name + workdir` | 找/建 | 自动新建 |
| `POST /v1/runs`(无 project 字段) | **不持久化**(向后兼容) | — |
| `POST /v1/plans body.project_id` | 显式必传 | (走原 PlanOrchestrator 路径) |
| `POST /v1/projects body.project_name + workdir` | 服务端按 name 自动建 | 始终新建(原行为) |

[情况 2] workdir 0 项目:
  当前工作目录下尚无项目。

## 4. /project 子命令(chat 内 9 个)

| 命令 | 功能 |
|------|------|
| `/project` 或 `/project list` | 列当前 workdir 所有项目(* 标记当前) |
| `/project current` | 显示当前活动项目元信息 |
| `/project info` | 当前项目元信息 + 对话条数 + 记忆条数 |
| `/project new <name>` | 新建并切换 |
| `/project switch <name|id>` | 切到本 workdir 内别的项目(不允许跨 workdir) |
| `/project rename <new>` | 重命名当前项目 |
| `/project delete <name|id>` | 删除项目(二次确认 + 级联删除对话/记忆/元数据) |
| `/project save <name>` | **救援命令**:无项目模式中途把当前对话保存为新项目 |
| `/project help` | 帮助 |

## 5. ProjectsStore 接口(`sdk/persistence/projects_store.go`)

```go
type ProjectsStore interface {
    Create(ctx, p *ProjectMeta) error
    Get(ctx, id string) (*ProjectMeta, error)
    FindByName(ctx, workdir, name string) (*ProjectMeta, error)
    ListByWorkdir(ctx, workdir string) ([]*ProjectMeta, error)
    UpdateLastActive(ctx, id string, t time.Time) error
    Rename(ctx, id, newName string) error
    Delete(ctx, id string) error
}

type ProjectMeta struct {
    ID           string
    Workdir      string
    Name         string
    CreatedAt    time.Time
    LastActiveAt time.Time
    Metadata     map[string]string
}

func GenerateProjectID() string  // 12 位 hex 随机
```

提供 SQLite + 内存两种实现。

## 7. 默认行为说明 — 为什么 run 默认不持久化

`brain run`(无 `--project`)和 `POST /v1/runs`(无 `project_id`/`project_name`)**默认不写项目持久化**。理由:

1. **向后兼容**:旧用户脚本期望 run 是一次性单次执行
2. **避免污染**:用户在 /tmp 跑测试不希望写入项目库
3. **明确意图**:持久化是有副作用的,要求显式声明

**chat 模式**强制弹选择器是因为 chat 是交互式高频使用,这是它"长期记忆"的核心用例。

## 8. ContextEngine 自动加载历史

当 `state.CurrentProject` 非空时,所有 `Orchestrator.delegateOnce` 在调 `ContextEngine.Assemble`(req)时,`req.ProjectID = CurrentProject.ID`,自动:

1. 调 `ProjectStore.LoadMessages(ProjectID, 20)` 拿最近 20 条对话
2. 调 `ProjectMemory.Summarize(ProjectID, 2000 token)` 拿记忆摘要
3. 注入为 `[项目记忆]` system 消息前置

详见 `35-Context-Engine详细设计.md` §5。
