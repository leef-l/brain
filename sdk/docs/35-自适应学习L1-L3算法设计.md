## 0. 四层学习模型

```
L3 LLM 记忆      ProjectMemory + UserPreference         (跨项目)
L2 序列学习      SequenceLearner + RecommendOrder        (任务流)
L1 统计 + Bandit BrainCapabilityProfile + RankBrains     (单步)
L0 brain 内特化  brains/<kind>/learner.go               (脑内)
```

L0 是各 brain 自己的学习闭环(quant 学交易策略,data 学数据质量阈值,等),不在 SDK 编排层。L1-L3 是 SDK 编排层共有的学习能力,**MACCS Wave 5 在 L1-L3 之上加了 5 项横向增强**(因果/迁移/主动/模式/Prompt)。

## 1. L1 — BrainCapabilityProfile + RankBrains(`learning.go`)

### 1.1 EWMAScore(`line 26`)

```go
type EWMAScore struct {
    Value float64  // 当前估值
    Alpha float64  // 平滑系数(默认 0.3)
}
func (e *EWMAScore) Update(newVal) { Value = (1-α)*Value + α*newVal }
```

> **EWMA**(指数加权移动平均):新值权重 α=0.3,旧值权重 0.7。响应快但平滑,标准 bandit 推荐。

### 1.2 BrainCapabilityProfile(`line 59`)

```go
type BrainCapabilityProfile struct {
    BrainKind   agent.Kind
    TaskTypes   map[string]*TaskTypeScore  // 按任务类型分桶
    LastUpdated time.Time
}

type TaskTypeScore struct {
    SuccessCount int
    FailCount    int
    Accuracy     EWMAScore  // 成功率
    Speed        EWMAScore  // 响应速度(秒)
    Cost         EWMAScore  // 成本估算
    Stability    EWMAScore  // 稳定性(标准差倒数)
}
```

每个 (brain, taskType) 二元组维护 4 个 EWMA 指标。

### 1.3 RankBrains 公式

```go
func ComputeWeights(policy WeightPolicy) (wAcc, wSpd, wCst, wStab float64)
score = Accuracy*wAcc + (1/Speed)*wSpd + (1/Cost)*wCst + Stability*wStab
       × WilsonConfidence(n)   // 样本不足时打折
```

> **WilsonConfidence**(`line 106`):n 越少,折扣越大。`WilsonConfidence(10) ≈ 0.61`,`WilsonConfidence(100) ≈ 0.85`。避免"试 1 次成功 = 100% 准确率"的偏差。

## 2. L2 — SequenceLearner(`learning.go:150`)

```go
type SequenceLearner struct {
    sequences []TaskSequenceRecord
    mu        sync.RWMutex
}

type TaskSequenceRecord struct {
    Steps      []TaskStep
    Success    bool
    Duration   time.Duration
    RecordedAt time.Time
}

type TaskStep struct {
    BrainKind   agent.Kind
    TaskType    string
    Outcome     string         // "success"/"fail"/"partial"
    DurationSec float64
    // MACCS 5.1 因果学习用:观察混杂因子
    ContextSize int
    Complexity  string
    TimeBucket  string
    ProjectKind string
}

RecordSequence(record)        // 记录一次完整任务序列
RecommendOrder(steps)         // 根据历史推荐执行顺序
```

### 2.1 RecommendOrder 算法

基于历史 sequence 中相似 step 集合的成功率,排序时把"通常先做的、成功率高的"放前。**不破坏拓扑约束**——在 ExecuteTaskPlan 同 layer 内重排同层任务。

### 2.2 接入路径

`Orchestrator.recordDelegateOutcome` → `SequenceLearner.RecordSequence` 同时把 step 喂给 `CausalLearner.Observe` 和 `LearnRelations`。

## 5. MACCS 5.1 — 因果学习(`causal_learning.go`)

### 5.1.1 接口

```go
type CausalLearner interface {
    Observe(obs CausalObservation)
    LearnRelations() []CausalRelation
    QueryCause(effect) []CausalRelation       // 这个结果是谁导致的
    QueryEffect(factor, value) []CausalRelation // 这个因子会导致什么
    Counterfactual(obs, factor, newValue) float64  // 反事实:换个值会怎样
    Suggest(target) []CausalFactor             // 想要某结果应该控制哪些因子
}

type CausalObservation struct {
    Factors map[string]string // 4 个混杂因子(brain/contextSize/complexity/projectKind)
    Outcome string             // "success"/"fail"
    Strength float64
}

type CausalRelation struct {
    Cause      CausalFactor
    Effect     string
    Strength   float64       // 关系强度
    Confidence float64       // 置信度
    Direction  string        // "positive" / "negative"
}
```

### 5.1.2 接入路径(0139b5e Wave 7)

```go
// orchestrator.go:1591 resolveTargetKind
// 选 brain 时三因子加权:
combined = capScore * 0.4
        + learnScore * 0.25
        + causalScore * 0.35    // 0139b5e 把因果权重 0.2 → 0.35

// causalScore 来自 brainCausalScore(causal, kind):
//   QueryEffect("brain", kind) 找 effect="success" 的关系
//   按 strength × confidence 加权
//   取所有命中关系的最大 raw ∈ [-1, 1] → 线性映射到 [0, 1]
```

> **0139b5e 升级理由**:原 0.2 太弱,被表观相关性淹没。0.35 让因果信号在评分接近时主导路由,剔除"高成功率是因为该 brain 历史接的任务都简单"这类混杂相关。

### 8.2 Extract 算法(`Extract`,line 107)

```

  统计共同 tags(commonElements)
```

## 11. 全链路示例

```
用户 prompt → Plan
  PlanOrchestrator.ExecuteProject:
    1. 复杂度预估 ComplexityEstimator
       → learning(L1 历史) → transfer(5.2 跨项目) → heuristic(规则) 三段降级
    2. 元认知反思 MetaCognitive.Reflect
       → 取上轮 lesson(0139b5e Lessons 反馈)
    3. 记忆检索 MemoryRetriever.Retrieve
       → 4 维加权 top-N 写入 reflection.Recommendations
    4. SmartScheduler 分层 + ConflictDetector 冲突检测

  Orchestrator.delegateOnce:
    1. resolveTargetKind 路由(0139b5e 公式)
       combined = capScore*0.4 + learnScore*0.25 + causalScore*0.35
       (其中 5% 概率走 active explore 探索高不确定 brain)
    2. ContextEngineWithMemory.Assemble
       → 加载 ProjectMemory + lesson + pattern + 5.4 抽取的 patterns
       → 注入 [项目记忆] system 消息
    3. AdaptivePromptManager.SelectVariant
       → A/B 变体作 L1 system block 前置(cache=true)
    4. LLM 调用

  recordDelegateOutcome:
    1. SequenceLearner.RecordSequence(L2)
       → CausalLearner.Observe + LearnRelations(5.1)
       → 4 混杂因子 contextSize/complexity/timeBucket/projectKind
    2. assessActiveLearning(5.3)
       → AssessUncertainty
       → 不确定 → EventBus.Publish("brain.feedback.requested")
       → consumeFeedbackRequests goroutine 把请求作 lesson 写 ProjectMemory

完成后:
  runPatternExtraction(5.4)
    → ExperienceStore.Save → Extract → AddPattern → Memory.Store
```

## 12. 关键事实速查

- **路由公式**:`combined = cap*0.4 + learn*0.25 + causal*0.35`
- **active 探索率**:5%(`activeExplorationEpsilon = 0.05`)
- **lesson 阈值**:0.3(0139b5e 从 0.5 降下来扩大反馈进入)
- **EWMA α**:0.3
- **Wilson 折扣**:n=10 → 0.61,n=100 → 0.85
- **5 + 1 项 v2 升级**:5.1 因果 / 5.2 迁移 / 5.3 主动 / 5.4 模式 / 5.5 Prompt + 5.6 画像
