## 1. 概述

v3 的每个专精大脑（code、browser、verifier、data、quant、fault 以及第三方 brain）都支持两种运行模式：

| 模式 | 启动方式 | 通信 | 场景 |
|------|---------|------|------|
| **本地 sidecar** | Kernel fork 子进程 | stdin/stdout | 默认，单机部署 |
| **远程网络服务** | 手动启动 `--listen` | HTTP + WebSocket | GPU 集群、团队共享、云工作站 |

两种模式共用同一个 `BrainHandler` 实现，业务逻辑零改动。

---

## 2. 架构

### 2.1 本地模式（默认）

```
┌──────────────────────────────┐
│ 同一台机器                     │
│                               │
│  Kernel ◄─── stdio ───► brain-code（子进程）│
│                               │
└──────────────────────────────┘
```

Kernel 通过 `ProcessRunner` fork sidecar 二进制，stdin/stdout 上跑 Content-Length framed JSON-RPC。

### 2.2 远程模式

```
你的机器                              远程服务器
┌─────────────┐    network      ┌──────────────────────┐
│   Kernel    │◄── WebSocket ──►│ brain-code            │
│             │    BidirRPC     │ --listen :8080        │
│ RemotePool  │                 │ ├─ POST /rpc          │
│             │                 │ ├─ GET  /ws           │
│             │                 │ └─ GET  /health       │
└─────────────┘                 └──────────────────────┘
```

Kernel 通过 `RemoteBrainPool` 连接到远程 brain 的 HTTP+WebSocket 端点。

## 3. 快速上手

### 3.1 启动远程 brain

```bash
# 在 GPU 服务器上
brain-code --listen :8080

# 输出：
# brain-code sidecar v0.6.0 listening on :8080 (network mode)
```

### 3.2 配置 Kernel 连接

在本地机器的 `~/.brain/config.json` 中添加：

```json
{
  "remote_brains": [
    {
      "kind": "code",
      "endpoint": "http://gpu-server:8080",
      "api_key": "your-secret-token",
      "timeout": "30s",
      "auto_start": true
    }
  ]
}
```

### 3.3 启动 Kernel

```bash
brain serve --listen :7701
```

Kernel 启动时自动：
1. 连接远程 brain 的 `/rpc` 端点做 health check
2. 升级为 WebSocket `/ws` 建立双向 RPC 会话
3. 注册到 `RemoteBrainPool`，后续 Orchestrator 可委派任务

## 4. 通信协议

### 4.1 HTTP JSON-RPC（/rpc 端点）

单次请求/响应，无反向调用能力。适合简单查询。

**请求：**
```http
POST /rpc HTTP/1.1
Content-Type: application/json
Authorization: Bearer sk-xxx

{"jsonrpc":"2.0","id":1,"method":"brain/execute","params":{"instruction":"写一个 hello world"}}
```

**响应：**
```json
{"jsonrpc":"2.0","id":1,"result":{"status":"completed","summary":"已完成","turns":3}}
```

支持的方法：

| 方法 | 说明 |
|------|------|
| `health.ping` | 健康检查 |
| `initialize` | 协议握手 |
| `tools/list` | 列出支持的工具 |
| `tools/call` | 调用单个工具 |
| `brain/execute` | 执行任务（单次） |
| `brain/metrics` | 获取学习指标 |

### 4.2 WebSocket BidirRPC（/ws 端点）

全双工双向 RPC，支持反向调用。这是 Orchestrator 委派的默认通信方式。

**连接升级：**
```
GET /ws HTTP/1.1
Upgrade: websocket
Connection: Upgrade
Authorization: Bearer sk-xxx
```

**消息格式（WebSocket text message）：**
```json
{
  "content_length": 123,
  "content_type": "application/vscode-jsonrpc; charset=utf-8",
  "body": {"jsonrpc":"2.0","id":"k:1","method":"brain/execute","params":{...}}
}
```

与 stdio 模式使用完全相同的 JSON-RPC 协议和帧格式，只是传输层从 stdio 换成了 WebSocket。

### 4.3 反向调用

远程 brain 可以通过同一个 WebSocket 连接**反向调用 Kernel**：

| 反向方法 | 说明 |
|---------|------|
| `llm.complete` | 请求 Kernel 代理 LLM 调用 |
| `subtask.delegate` | 请求 Kernel 委派子任务到其他 brain |
| `specialist.call_tool` | 请求 Kernel 调用其他 brain 的工具 |

这使得远程 brain 可以运行完整的 Agent Loop：

```
远程 brain                         Kernel
├─ 收到 brain/execute              │
├─ 需要 LLM 推理                   │
│  llm.complete ────────────────► │ 代理到配置的 LLM provider
│  ◄──────────────── LLM 结果      │
├─ LLM 返回 tool_use               │
├─ 本地执行工具                      │
├─ 需要再次 LLM 推理               │
│  llm.complete ────────────────► │
│  ◄──────────────── LLM 结果      │
├─ LLM 返回 end_turn               │
└─ 返回最终结果 ─────────────────► │
```

## 5. 使用场景

### 5.1 GPU 推理集群

将 quant brain 部署到有 GPU 的服务器：

```bash
# GPU 服务器
brain-quant-sidecar --listen :8081
```

```json
{
  "remote_brains": [
    {"kind": "quant", "endpoint": "http://gpu-01:8081"}
  ]
}
```

### 5.2 团队共享 code brain

多个开发者共享一个 code brain 实例：

```bash
# 共享服务器
brain-code --listen 0.0.0.0:8080
```

每个开发者配置：
```json
{
  "remote_brains": [
    {"kind": "code", "endpoint": "http://team-server:8080", "api_key": "team-token"}
  ]
}
```

**注意**：团队共享模式下，远程 brain 的工具在远程执行。需要确保 workdir 隔离——每个用户的请求应带不同的 `execution.workdir`。

### 5.3 混合部署

本地 brain + 远程 brain 混合：

```json
{
  "remote_brains": [
    {"kind": "quant", "endpoint": "http://gpu-01:8081"},
    {"kind": "data",  "endpoint": "http://data-server:8082"}
  ]
}
```

code/browser/verifier 仍然在本地运行（默认 stdio 模式），quant/data 走远程。Orchestrator 自动选择本地或远程 pool。

## 6. 安全

### 6.1 认证

远程 brain 支持 Bearer token 认证：

```json
{"api_key": "sk-your-secret-token"}
```

所有 HTTP 和 WebSocket 请求都带 `Authorization: Bearer sk-xxx` 头。

### 6.2 TLS

生产环境建议使用 HTTPS：

```json
{"endpoint": "https://brain.example.com:8443"}
```

WebSocket 自动升级为 `wss://`。

### 6.3 熔断保护

`RemoteBrainPool` 内置熔断器：
- 连续 3 次失败 → 熔断（拒绝请求 30 秒）
- 半开状态允许试探
- 连续 2 次成功 → 恢复

### 6.4 降级

当 WebSocket 不可用时，自动降级为 HTTP-only 模式（无反向调用）。降级后仍支持 `brain/execute` 单次调用，但不支持完整 Agent Loop。

## 7. 健康检查

### 7.1 GET /health

```bash
curl http://brain-server:8080/health
```

```json
{
  "status": "ok",
  "kind": "code",
  "version": "0.6.0"
}
```

### 7.2 JSON-RPC health.ping

```bash
curl -X POST http://brain-server:8080/rpc \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"health.ping"}'
```

```json
{"jsonrpc":"2.0","id":1,"result":{"status":"ok"}}
```

### 7.3 auto_start

配置 `"auto_start": true` 时，Kernel 启动时自动对远程 brain 做 health check。失败时打印警告但不阻塞启动。

## 8. 监控

### 8.1 Dashboard

Dashboard 的 `/v1/dashboard/brains` 端点显示远程 brain 状态：

```json
{
  "brains": [
    {"kind": "code", "running": true, "binary": "http://gpu-server:8080"},
    {"kind": "quant", "running": false, "binary": "http://gpu-01:8081"}
  ]
}
```

### 8.2 brain/metrics

Orchestrator 定期调用远程 brain 的 `brain/metrics` 方法，获取 L0 学习指标：

```json
{
  "brain_kind": "code",
  "task_count": 42,
  "success_rate": 0.85,
  "avg_latency_ms": 1200,
  "confidence_trend": 0.3
}
```

这些指标喂入 L1 能力画像，影响后续 brain 选择排名。

## 9. 端点一览

| 端点 | 方法 | 用途 | 模式 |
|------|------|------|------|
| `/health` | GET | 健康检查 | HTTP |
| `/rpc` | POST | 单次 JSON-RPC | HTTP |
| `/ws` | GET | WebSocket 双向 RPC | WebSocket |

每个端点的行为与本地 stdio 模式完全一致——相同的 JSON-RPC 方法、相同的请求/响应结构、相同的协议语义。唯一的区别是传输层。

## 10. 故障排除

### 连接失败

```
remote pool: health check failed for code at http://server:8080: ...
```

检查：远程 brain 是否已启动？端口是否可达？防火墙？

### WebSocket 降级

```
remote pool: code ws unavailable (...), falling back to HTTP-only
```

远程 brain 的 `/ws` 端点不可用。反向调用（llm.complete）将不可用。检查 WebSocket 端点和代理配置。

### 熔断触发

```
circuit breaker open for code
```

远程 brain 连续失败已触发熔断。等待 30 秒后自动恢复试探。检查远程 brain 进程是否存活。
