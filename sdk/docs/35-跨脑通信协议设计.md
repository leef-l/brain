## 0. 协议层级

```
应用层:Method 字符串 + JSON 参数(本文档主体)
RPC 层:JSON-RPC 2.0(request/response/notification)
帧格式:Content-Length 头 + UTF-8 JSON body
传输层:stdio(本地 sidecar) / HTTP(远程同步) / WebSocket(远程双向)
```

> **三种 transport 共用同一 Method 名空间**——业务代码不感知传输方式。

## 2. JSON-RPC 帧格式(`sdk/protocol/frame.go`)

### 2.1 头格式

```
Content-Length: <byte_count>\r\n
\r\n
<UTF-8 JSON body>
```

> 与 LSP / Claude Code MCP 同款。`Content-Length` 必须严格匹配 body 字节数。

### 2.2 JSON-RPC 2.0 消息类型

| 类型 | 字段 |
|------|------|
| Request | `{jsonrpc:"2.0", id, method, params}` |
| Response(成功) | `{jsonrpc:"2.0", id, result}` |
| Response(失败) | `{jsonrpc:"2.0", id, error:{code, message, data}}` |
| Notification | `{jsonrpc:"2.0", method, params}` (无 id,无回包) |

### 2.3 frameWriter 并发安全(F-1 修复)

```go
type frameWriter struct {
    w      io.Writer
    mu     sync.Mutex
    closed bool
}
func (fw *frameWriter) WriteFrame(...) error {
    fw.mu.Lock()
    defer fw.mu.Unlock()
    if fw.closed { return ErrClosed }
    // 写头 + body
}
```

> **历史 race**:`bidirRPC.readLoop()` close channel 时 `Notify()` 可能正在发送。已用 `mu.Lock()` + `closed` flag 修复(2026-04-24 核查通过)。

## 3. BidirRPC 双向 RPC(`sdk/protocol/rpc.go`)

```go
type BidirRPC interface {
    Call(ctx, method, params) (json.RawMessage, error)  // 同步请求
    Notify(method, params) error                         // 单向通知
    HandleMethod(method, handler)                        // 注册 handler
    Close() error
}
```

### 3.1 readLoop(`rpc.go:readLoop`)

```
持续读 frame:
  parse JSON-RPC message
  if 是 response: 找 pending 表里的 chan,送结果
  if 是 request:  调 HandleMethod 注册的 handler,回 response
  if 是 notification: 调 handler,无回包
EOF / err: 关闭 pending 表所有 chan,退出
```

### 3.2 Notify 非阻塞设计

`Notify` 写入 frameWriter,不等回包,**用于高频 stream/observability 类**(brain/stream/write、trace.emit、progress/report 等)。

## 5. MACCS 5.3 — brain.feedback.requested(EventBus,非 RPC)

### 5.1 不是 RPC 而是内部事件

> 注意:`brain.feedback.requested` 是 **kernel 内部 EventBus 事件**,不走 protocol RPC。它由 ActiveLearner.AssessUncertainty 命中阈值时由 Orchestrator 在内部 Publish,订阅者(PlanOrchestrator goroutine)在同进程消费。

```go
// sdk/events/event_bus.go(简化)
type EventBus interface {
    Publish(event Event)
    Subscribe(eventType string) <-chan Event
}

// orchestrator.go:1813 assessActiveLearning
score := learner.Active().AssessUncertainty(taskID, kind, successRate, dataPoints)
if score.Score >= threshold:
    eventBus.Publish(Event{
        Type: "brain.feedback.requested",
        Payload: FeedbackRequest{...},
    })
```

### 5.2 订阅方(0139b5e Wave 7 修复)

```go
// plan_orchestrator.go:159 consumeFeedbackRequests
go func() {
    ch := eventBus.Subscribe("brain.feedback.requested")
    for ev := range ch:
        memory.Store(MemoryEntry{ Type: MemoryLesson, ... })
}()
```

> **修复说明**:0139b5e 之前**只有 Publisher,没有 Subscriber**——事件被发出但无人接,5.3 的反馈闭环断裂。Wave 7 修复:订阅 goroutine 把每条反馈请求作为 lesson 写入 ProjectMemory,下轮 plan 经 MemoryRetriever 读到。

## 9. 流式输出实现(MethodLLMStream + LLMStreamDelta)

```
sidecar → host:
  POST llm.stream { messages, model, ... }      // request
  ← response { stream_id }                       // host 立即 ack

host → sidecar(streaming notification):
  Notify llm/stream/delta { stream_id, content_block, delta }
  ...(多次 delta)
  Notify llm/stream/delta { stream_id, message_stop: true }
```

> sidecar 端的 `streaming.go` 把这些 delta 还原为完整响应,边收边 yield。2026-04-27 c4fe85b 修复:从聚合式伪流式升级为真正的跨端 Notify 实时推送。

## 10. 全链路示例(简化版)

```
[Run 开始]
  CLI/HTTP API → Orchestrator.Delegate
  → BrainPool.GetBrain(kind) 拿到 BidirRPC
  → 通过 RPC 发 subtask.delegate{...}

[sidecar 内部]
  → ProcessRunner 启动 sidecar 进程,stdio JSON-RPC
  → sidecar Initialize 握手
  → 接 subtask.delegate
  → 内部跑 Runner.Execute (loop):
      → 反向 llm.complete 或 llm.stream 让 host 调 LLM
      → host Notify llm/stream/delta(多次)
      → sidecar 重组 response,提取 tool_use
      → 反向 specialist.call_tool 让 host 跑工具
      → host 跑工具,返回结果
      → sidecar Notify progress/report(进度)
      → sidecar Notify trace.emit / audit.emit(可选)
      → 最终 task_complete tool 触发 run 终止
  → sidecar 返回 subtask.delegate response

[host 处理结果]
  → recordDelegateOutcome
  → SequenceLearner / CausalLearner 学习
  → assessActiveLearning → EventBus.Publish("brain.feedback.requested")
  → consumeFeedbackRequests → ProjectMemory.Store
```
