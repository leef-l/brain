# 29. 第三方专精大脑开发指南

> **版本**: v3.0(全量对齐 sdk/sidecar + 模板 + 双模式 + ModelRouter)
> **更新日期**: 2026-05-02
> **目标**: 让第三方开发者独立实现并发布自己的 Specialist Brain,通过 BrainKernel 的 sidecar 协议被主进程或 central brain 调用
> **前置阅读**: `32-v3-Brain架构.md` / `20-协议规格.md` / `35-跨脑通信协议设计.md`

---

## 0. 一句话定义

> 第三方专精大脑 = 一个 Go 二进制 + 一份 `brain.json` manifest。
> 实现 `sidecar.BrainHandler` 接口,跑 `sidecar.Run(handler)`,放到 `$GOPATH/bin/` 即可被 Kernel 自动发现并调用。

## 1. 最小可用 brain(80 行代码)

### 1.1 项目结构

```
my-specialist/
├── go.mod
├── brain.json                 manifest:声明 kind / capabilities / 版本
└── cmd/
    └── main.go                sidecar 入口
```

### 1.2 brain.json

```json
{
  "schema_version": 1,
  "kind": "my-specialist",
  "name": "My Specialist Brain",
  "brain_version": "0.1.0",
  "capabilities": [
    "function.summarize",
    "domain.text"
  ],
  "runtime": {
    "type": "native",
    "entrypoint": "brain-my-specialist-sidecar"
  },
  "policy": {
    "approval_class": "workspace-write"
  }
}
```

### 1.3 main.go

```go
package main

import (
    "context"
    "encoding/json"
    "fmt"
    "os"

    "github.com/leef-l/brain/sdk/agent"
    "github.com/leef-l/brain/sdk/sidecar"
)

type Handler struct{}

func (h *Handler) Kind() agent.Kind { return "my-specialist" }
func (h *Handler) Version() string  { return "0.1.0" }
func (h *Handler) Tools() []string  { return []string{"my-specialist.summarize"} }

func (h *Handler) HandleMethod(ctx context.Context, method string, params json.RawMessage) (interface{}, error) {
    switch method {
    case "brain/execute":
        return map[string]interface{}{
            "status":  "completed",
            "summary": "done",
            "turns":   1,
        }, nil
    case "tools/call":
        var req struct {
            Name      string          `json:"name"`
            Arguments json.RawMessage `json:"arguments"`
        }
        json.Unmarshal(params, &req)
        return map[string]interface{}{
            "tool":   req.Name,
            "output": "Hello from my-specialist!",
        }, nil
    default:
        return nil, sidecar.ErrMethodNotFound
    }
}

func main() {
    if err := sidecar.Run(&Handler{}); err != nil {
        fmt.Fprintln(os.Stderr, err)
        os.Exit(1)
    }
}
```

### 1.4 编译与安装

```bash
go build -o $GOPATH/bin/brain-my-specialist-sidecar ./cmd
cp brain.json $GOPATH/bin/brain-my-specialist.brain.json
```

> **二进制命名规则**:`brain-{kind}-sidecar` (Kernel 通过 `binResolver` 在 `$GOPATH/bin/` 探测;详见 `35-BrainPool实现设计.md` §1)。

### 1.5 验证接入

```bash
brain doctor                 # 应该列出 my-specialist 可用
brain run --brain my-specialist "你好,做个测试"
```

## 2. sdk/sidecar 包 API 参考

### 2.1 BrainHandler 接口(`sidecar/sidecar.go:26`)

```go
type BrainHandler interface {
    Kind() agent.Kind                                                                      // brain 角色
    Version() string                                                                       // sidecar 自身版本
    Tools() []string                                                                       // 支持的工具名
    HandleMethod(ctx, method string, params json.RawMessage) (interface{}, error)         // RPC 派发
}

var ErrMethodNotFound = fmt.Errorf("method not found")  // 未知 method 时返回此错误
```

`sidecar.Run(handler)` 自动处理:
- `initialize` 握手
- `$/shutdown` 优雅关停
- `heartbeat` 保活
- `tools/list` 工具查询
- 信号处理 (SIGTERM / SIGINT)

业务方法 (`brain/execute`、`tools/call`、自定义 method) 走 `HandleMethod`。

### 2.2 RichBrainHandler — 反向 RPC 调用 Kernel(`sidecar.go:62`)

如果 brain 需要让 Kernel 帮自己调 LLM 或工具,实现 `RichBrainHandler`:

```go
type KernelCaller interface {
    CallKernel(ctx, method string, params, result interface{}) error  // 同步 RPC
    NotifyKernel(ctx, method string, params interface{}) error          // 单向通知
}

type RichBrainHandler interface {
    BrainHandler
    SetKernelCaller(caller KernelCaller)  // 框架在握手后注入
}
```

### 2.3 反向 RPC 典型用法

```go
type Handler struct {
    caller sidecar.KernelCaller
}

func (h *Handler) SetKernelCaller(c sidecar.KernelCaller) { h.caller = c }

func (h *Handler) summarize(ctx context.Context, text string) (string, error) {
    var resp llm.ChatResponse
    err := h.caller.CallKernel(ctx, "llm.complete", llm.ChatRequest{
        Model: "claude-haiku-4-5-20251001",
        Messages: []llm.Message{{
            Role:    "user",
            Content: []llm.ContentBlock{{Type: "text", Text: "总结: " + text}},
        }},
    }, &resp)
    if err != nil { return "", err }
    return resp.Content[0].Text, nil
}
```

可调的 host 方法见 `35-跨脑通信协议设计.md` §1:`llm.complete` / `llm.stream` / `tool.invoke` / `specialist.call_tool` / `progress/report` / `trace.emit` / `audit.emit` 等。

## 3. 模板生成器 — `sdk/template/`

仓库自带模板:

```
sdk/template/
├── README.md                  使用说明
├── brain.json.tmpl            manifest 模板
├── main.go.tmpl               入口模板
└── handler.go.tmpl            完整 Handler 示例 (含 brain/execute + tools/call)
```

模板变量:`{{.Kind}}` / `{{.Name}}`。第三方开发者可:

```bash
# 假设有个工具能渲染模板
go run scripts/scaffold.go --kind=my-specialist --name="My Specialist"
# 在 examples/ 下生成完整可编译骨架
```

(scaffold 脚本由用户自行实现,本仓库未包含。模板本身可被任何 Go template 引擎渲染。)

## 4. brain.json Manifest 详细字段(`33-Brain-Manifest规格.md`)

```json
{
  "schema_version": 1,                                          必填:当前固定为 1
  "kind": "my-specialist",                                       必填:brain 角色,全局唯一
  "name": "My Specialist Brain",                                 必填:可读名称
  "brain_version": "0.1.0",                                      必填:sidecar 自身版本
  "capabilities": [                                              建议:能力标签 (35-BrainCapability)
    "function.summarize",
    "function.translate.v2",
    "domain.text",
    "resource.network",
    "mode.stream"
  ],
  "runtime": {
    "type": "native",                                           native (Go 二进制) / mcp / remote
    "entrypoint": "brain-my-specialist-sidecar",                native 时为二进制名
    "args": ["--mode", "production"],                           可选:启动参数
    "env": ["MY_API_KEY=xxx"]                                    可选:环境变量
  },
  "policy": {
    "approval_class": "workspace-write",                         trivial / notable / safety / critical
    "max_instances": 5,                                          MaxInstances:并发上限 (35-BrainPool)
    "auto_start": true                                           是否随 Kernel 自动启动
  },
  "models": {                                                    可选:推荐模型 (5.5 ModelRouter)
    "primary": "claude-sonnet-4-6",
    "fallback": "claude-haiku-4-5-20251001"
  }
}
```

## 5. 双模式接入(独立运行 + sidecar)

如果你的 brain 既要支持**作为 Kernel 的 sidecar**(被 stdio 调用) 又要**独立运行**(直接 CLI / HTTP),用 `data` / `quant` 的双模式架构:

```
brains/my-brain/
├── brain.json
├── main.go                    独立运行入口 (CLI / HTTP server)
├── learner.go                 (可选) L0 自适应学习
└── cmd/
    └── brain-my-brain-sidecar/
        └── main.go            sidecar 入口 (stdio JSON-RPC)
```

两份 main.go 共享 brain 业务逻辑,只在入口层区别:
- `cmd/main.go` (独立模式) 启动 HTTP server / 进入 REPL
- `cmd/brain-my-brain-sidecar/main.go` 调 `sidecar.Run(handler)`

`build-assets.bat` / `package.sh` 自动识别这种结构(Pattern 1 双模式 + Pattern 3 sidecar 子目录)。

## 6. 远程模式接入

如果你的 brain 不在用户机器上(比如部署在云端),可作为**远程 brain** 接入。详见 `37-远程专精大脑调用说明.md`。简化路径:

```yaml
# ~/.brain/config.json
remote_brains:
  - kind: "my-specialist"
    endpoint: "https://my-specialist.example.com/rpc"
    api_key: "${MY_SPECIALIST_API_KEY}"
    timeout: "30s"
    auto_start: false
```

服务端实现 HTTP/WebSocket BidirRPC,响应同样的 `brain/execute` / `tools/call` 等方法。Kernel 通过 `RemoteBrainPool` + `ServiceDiscovery` 自动发现并调用。

## 7. 模型差异化配置(MACCS 2.6 ModelRouter)

ModelRouter 让不同 brain 用不同 LLM 模型:

```yaml
# ~/.brain/config.json
maccs:
  model_router:
    enabled: true
    mappings:
      central:        "claude-opus-4-7"            # 长上下文 + 强推理
      code:           "claude-sonnet-4-6"          # 代码任务
      data:           "claude-haiku-4-5-20251001"  # 数据处理 (延迟敏感)
      verifier:       "claude-opus-4-7"            # 审核需要严密推理
      my-specialist:  "claude-sonnet-4-6"          # 第三方 brain 也参与映射
```

第三方 brain 通过 `brain.json` 的 `models` 字段声明默认模型;运行时 ModelRouter 优先用 config 的 mappings 覆盖。

## 8. 安全与权限

### 8.1 approval_class 级别(详见 `35-语义审批分级设计.md`)

| 级别 | 含义 | 例子 |
|------|------|------|
| `trivial` | 无副作用 | 只读查询、计算 |
| `notable` | 工作区内可逆操作 | 写文件、git commit |
| `safety` | 工作区外或不可逆 | 修改 home 配置、安装包 |
| `critical` | 系统级或外部影响 | 删根目录、对外发邮件、转账 |

### 8.2 file_policy / sandbox

第三方 brain 受 `file_policy` 全局规则约束(详见 `23-安全模型.md`)。即便 brain 自己代码有 `os.WriteFile`,在 `restricted` 模式下越权写也会被 sdk/security 拦截。

### 8.3 工具风险评估

`sdk/tool/risk.go` 维护工具风险等级。第三方 brain 注册的工具如果做敏感操作,**SHOULD** 在 `Tools()` 返回前调 `tool.RegisterRisk(toolName, RiskHigh)` 显式标注。

## 9. 学习层接入(可选 L0)

如果你的 brain 想接入 L0 自适应学习(领域内的本地学习),实现 `BrainLearner` 接口:

```go
// 见 sdk/kernel/learning.go 的 BrainLearner 类型
type BrainLearner interface {
    Adapt(observation Observation) error    // 接收一次执行的观察值
    Snapshot() LearnerSnapshot               // 导出学习状态
    Restore(snapshot LearnerSnapshot) error  // 从快照恢复
}
```

参考 `brains/{quant,code,browser,fault,verifier}/learner.go` 的实现。

L1-L3 是 SDK 编排层共有的学习能力,你的 brain **不需要**自己实现 L1-L3——只需要给 Kernel 上报指标(通过 `brain/metrics` notification)即可被 LearningEngine 学习。

## 12. 完整执行链路(开发者视角)

```
1. 用户跑 brain run --brain my-specialist "总结这段文本"
2. Kernel CLI/Orchestrator 解析:target_kind = "my-specialist"
3. BrainPool.AcquireBrain("my-specialist"):
   - 探测 $GOPATH/bin/brain-my-specialist-sidecar 存在 → 启动
   - stdio JSON-RPC 握手 (initialize)
4. Orchestrator.delegateOnce:
   - resolveTargetKind 走 capability 匹配:你的 capabilities 含 "function.summarize" → 命中
   - ContextEngine.Assemble → 注入项目记忆
   - ModelRouter.Resolve("my-specialist") → "claude-sonnet-4-6"
5. 发 RPC subtask.delegate{ task, model, session_id, ... }
6. 你的 Handler.HandleMethod("brain/execute", params):
   - 解析 task
   - (如需) 反向调 host: caller.CallKernel("llm.complete", ...)
   - 返回 { status: "completed", summary: "...", turns: N }
7. Orchestrator.recordDelegateOutcome:
   - SequenceLearner.RecordSequence (L2)
   - CausalLearner.Observe (5.1)
   - assessActiveLearning → EventBus brain.feedback.requested (5.3)
8. 用户看到结果
```

## 13. 常见陷阱

| 陷阱 | 修复 |
|------|------|
| 二进制名错 (`brain-my-specialist`) | 必须 `brain-my-specialist-sidecar` |
| brain.json 在错的位置 | 跟二进制同目录 (`$GOPATH/bin/`) |
| capabilities 漏声明 | 不声明则匹配不到任务 (35-BrainCapability §3) |
| HandleMethod 返回 nil error 但结果格式不对 | Kernel 期待 map 含 status/summary/turns 字段 |
| 长任务不发 progress/report | Kernel 不知道你在干什么,30 秒后 timeout 中断 |
| 反向调 LLM 没设 SetKernelCaller | RichBrainHandler 接口才会被注入 KernelCaller |
| stderr 大量输出 | sidecar stdout 是 RPC 通道,业务日志只能写 stderr;过多 stderr 让 Kernel 终端难以阅读 |
