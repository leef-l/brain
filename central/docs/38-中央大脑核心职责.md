# 中央大脑核心职责

> 监督者 · 审查者 · 复盘分析师 — 不参与交易决策

**上级文档**: [35-量化系统三脑架构总览](35-量化系统三脑架构总览.md)

---

## 一、核心定位

```
Central Brain ≠ 交易决策者
Central Brain = 监督者 + 审查者 + 协调者 + 复盘分析师
```

Central Brain 在量化系统中**不参与正常交易路径**。它只在以下场景被触发：

| 职责 | 触发条件 | 频率 |
|------|---------|------|
| LLM 交易审查 | 高风险信号 (NeedsReview) | ~几次/天 |
| 每日复盘 | UTC 0:00 定时 | 1 次/天 |
| 策略权重调整 | 复盘后 / 手动 | 1 次/天 |
| 异常处理 | 数据质量告警、账号故障 | 按需 |
| 多账号监控 | 定时轮询 | 每 5 分钟 |
| 紧急干预 | 极端行情、全局止损触发 | 极少 |
| 宏观事件解读 | 重大新闻/数据发布 | 按事件 |

## 二、职责边界（2026-04-25 更新）

Central Brain **不负责** plan 管理。以下工具已在 v3 中移除：
- ~~`central.plan_create`~~ — 空壳，已删除
- ~~`central.plan_update`~~ — 空壳，已删除

Central Brain 当前职责仅包含：
- `central.delegate` — 委派子任务给专业大脑
- `central.review_trade` — LLM 交易审查
- `central.daily_review` — 每日复盘
- `central.data_alert` — 数据质量告警响应
- `central.reject_task` — 拒绝无法处理的任务

Plan 生命周期管理由 CLI 层（`brain run`）直接维护，不经过 Central Brain。
