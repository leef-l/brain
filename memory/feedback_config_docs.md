配置文件（YAML/JSON）的每个字段都必须有详细的中文注释说明和使用示例。

**Why:** 用户明确要求"配置文件所有配置都要有详细说明和示例，代码和示例配置文件保持同步"。缺少注释的配置文件被视为不合格。

**How to apply:**
- 每个配置字段必须有：字段含义、默认值、取值范围、实际效果举例
- 每个配置段（section）需要提供至少一个完整的使用示例
- 新增代码字段时，必须同步更新 `~/.brain/quant-brain.yaml` 中的注释
- 百分比/基点等单位必须明确标注换算关系（如 "5 基点 = 0.05%"）
