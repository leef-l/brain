# 目录重组方案

> 按大脑维度组织 — 每个大脑一个目录，文档跟着代码走

---

## 一、当前问题

1. **SDK 和量化系统混在同一层** — 26 个顶级目录，无法一眼看出哪个是 SDK、哪个是大脑
2. **cmd/ 职责混乱** — 主 CLI（59 个 .go 文件）和 6 个 sidecar 入口堆在一起
3. **internal/ 归属不清** — strategy/risk/execution 是量化大脑的，pathglob 是 SDK 的，全混着
4. **文档和代码分离** — docs/ 集中放，找代码对应的设计要来回跳
5. **垃圾残留** — 旧项目目录、二进制、HTML 文件（已清理）

### 3.5 cmd/ 根文件处理

cmd/ 下的 59 个 .go 文件（chat_*.go、cmd_*.go、runtime_*.go 等）是主 CLI 的实现。它们：
- `package main`，直接在 cmd/ 下
- 包含 REPL、chat、slash 命令、工具注册等
- 需要保留，但应该整体移到 `cmd/brain/` 下
